Semantic and Style Elements

Semantic Elements

Elements in the DOM represent things, they have intrinsic meaning, also known as semantics. Elements, attributes and attribute values in HTML are defined to have certain semantics or meanings. In HTML there are semantic elements that can be used to define different parts of a web page. These definitions allow HTML processors, such as web browsers or search engines, to present and use documents and applications in a wide variety of contexts.

  • Semantic Elements - A semantic element clearly describes its meaning/content to both the browser and the developer; some semantic elements include: <main>, <summary> and <article>
  • Non-Semantic Elements - Non-semantic elements, tell/represent nothing about its content, some examples of non-semantic elements are: <div> and <span>
Authors must not use elements, attributes or attribute values for purposes other than their appropriate intended semantic purpose, as doing so prevents software from correctly processing the page. According to the W3C: "A semantic Web allows data to be shared and reused across applications, enterprises, and communities."

Block and Inline Elements

Every HTML element has a default display value, depending on what type of element it is; there are two display values, block and inline:

  • Block-level Elements - A block-level element always starts on a new line, and browsers automatically add some space (a margin) before and after the element. A block-level element always takes up the full width available, as it stretches out to the left and right as far as it can. The block-level elements in HTML are the following: <address> <article> <aside> <blockquote> <canvas> <dd> <div> <dl> <dt> <fieldset> <figcaption> <figure> <footer> <form> <h1>-<h6> <header> <hr> <li> <main> <nav> <noscript> <ol> <p> <pre> <section> <table> <tfoot> <ul> <video>
  • Inline Elements - An inline element does not start on a new line, and will only take up as much width as necessary; it cannot contain a block-level element. The inline elements in HTML are the following: <a> <abbr> <acronym> <b> <bdo> <big> <br> <button> <cite> <code> <dfn> <em> <i> <img> <input> <kbd> <label> <map> <object> <output> <q> <samp> <script> <select> <small> <span> <strong> <sub> <sup> <textarea> <time> <tt> <var>

h1, h2, h3, h4, h5 and h6 Elements - <h1> <h2> <h3> <h4> <h5> <h6>

These elements represent headings for their sections, and have a rank (heading level) given by the number in their name; the level corresponds to levels of nested sections. The <h1> element defines the most important heading, and <h6> the least; browsers automatically add whitespace (margin) before and after a heading.
Headings Are Important - Users often skim a page by its headings, and search engines use the headings to index the structure and content of web pages; is important to use headings to show the document structure. User agents are encouraged to expose page outlines to users to aid in navigation, this is especially true for non-visual media, e.g. screen readers. Use HTML headings for headings only, don't use them to make text big or bold.

hgroup Element - <hgroup>

Element is used to group a set of <h1>–<h6> elements when the heading has multiple levels, such as subheadings, alternative titles or taglines. The element may be used to group an <h1>–<h6> element with one or more <p> elements containing content representing a subheading, alternative title or tagline. Other <h1>–<h6> elements of heading content in the <hgroup> element indicate subheadings, subtitles or alternative titles.

p Element - <p>

Element represents a paragraph, a paragraph always starts on a new line and usually is a block of text; browsers add some whitespace (margin) before and after a paragraph.

hr Element - <hr>

Element defines a thematic break in an HTML page, is used to separate content or define a change in an HTML page - the HTML Horizontal Rule.

main Element - <main>

Element specifies the main content of a document, each <main> element must be a hierarchically correct main element. A hierarchically correct main element is one whose ancestor elements are limited to html, body, div, form (without an accessible name) and autonomous custom elements. The element should not be a descendant of: <article>, <aside>, <footer>, <header> or <nav>. A document must not have more than one <main> element that does not have the hidden attribute specified. The content inside the <main> element should be unique to the document, and should not contain any content repeated across documents such as sidebars, navigation links, copyright information, site logos and search forms.

header Element - <header>

Element represents a group of introductory or navigational aids, and is intended to usually contain the section's heading, but is not required. The <header> element can also be used to wrap a section's table of contents, authorship information, a search form or any relevant logos.

footer Element - <footer>

Element represents a footer for its nearest ancestor sectioning content or sectioning root element. A footer typically contains information about its section such as who wrote it, links to related documents, copyright data and the like. When the <footer> element contains entire sections, they represent appendices, indexes, long colophons, verbose license agreements and other such content.

section Element - <section>

Element represents a section in a document, a thematic grouping of content; typically with a heading. A general rule is that the section element is appropriate only if the element's contents would be listed explicitly in the document's outline. Examples of sections would be chapters, various tabbed pages in a tabbed dialog box or the numbered sections of a thesis. A web site's home page could be split into sections for an introduction, news items and contact information.

article Element - <article>

Element defines an article or self-contained independent item of content. When article elements are nested, the inner article elements represent articles that are in principle related to contents of the outer article.

aside Element - <aside>

Element defines content that consists of content tangentially related to the content around the element's content, and which could be considered separate from that content. Can be used for typographical effects like pull quotes or sidebars for advertising, groups of nav elements and other content considered separate from the main content.

Interactive Elements

details Element - <details>

Element represents a disclosure widget from which the user can obtain additional information or controls, that the user can view or hide; by default the widget is closed. When open it expands and displays the content within, any sort of content can be put inside. The <summary> element child of the element, if any, represents the summary or legend of the details; the rest of element's contents represents the additional information or controls.

details Attributes
open
The open content attribute is a boolean attribute, if present, indicates that both the summary and additional information is to be shown to the user: <details open>

summary Element - <summary>

Element represents a summary caption or legend for the rest of the contents of the summary element's parent details element. The tag defines a visible heading for the details element, the heading can be clicked to view/hide details; it should be the first child element of the details element.

dialog Element - <dialog>

Element represents a transitory part of an application, in the form of a small window (dialog box), which the user interacts with to perform a task or gather information. Once the user is done, the dialog can be automatically closed by the application, or manually closed by the user. Especially for modal dialogs (a familiar pattern across applications), authors should ensure dialogs behave familiar to users of non-web applications. Authors should use the autofocus attribute on the descendant element of the dialog that the user is expected to immediately interact with after the dialog opens. Another important aspect of user behavior is whether dialogs are scrollable or not, in some cases, overflow (thus scroll-ability) cannot be avoided, e.g., when caused by user's high text zoom settings. Adding large text nodes directly to dialog elements is particularly bad, as it is likely to cause the dialog element to overflow - are best off avoiding them.

dialog Attributes
open
The open attribute is a boolean attribute, when specified indicates the dialog element is active and that the user can interact with it: <dialog open>

Non-Semantic Elements

div Element - <div>

Element defines a section in a document, it has no special meaning, but represents its children; it is used as a container for HTML elements. By default, browsers place a line break before and after the element; are strongly encouraged to view as an element of last resort, when no other element is suitable. Use of more appropriate elements instead of the <div> element leads to better accessibility for readers and easier maintainability for authors. On the other hand, <div> elements can be useful for stylistic purposes or to wrap paragraphs within a section that are annotated a similar way. Is then styled with CSS or manipulated with JavaScript, it is easily styled by using the class or id attributes.

span Element - <span>

Element defines a section in a document, and it doesn't mean anything on its own, but represents its children. Can be useful when used together with the global attributes, and the <span> tag is also easily styled by CSS or manipulated with JavaScript using class or id attributes. Is much like the <div> element, but <div> is a block-level element, and <span> an inline element; it is an inline container used to mark up part of a text or document.

Text Formatting and Semantics

HTML contains several elements for defining text with a special meaning. Phrasing content is the text of the document, as well as elements that mark up that text at the intra-paragraph level. Most elements categorized as phrasing content, can only contain elements that are themselves categorized phrasing content.

b Element - <b>

Element specifies bold text, and is a text to which attention is being drawn; such as key words, product names, article lede or actionable words in text-driven software.

bdi Element - <bdi>

Element isolates a part of text that might be formatted in a different direction from other text outside it, BDI stands for "Bi-Directional Isolation". This element is especially useful when embedding user-generated content, with an unknown directionality. The global dir attribute, defaults to auto on this element, and never inherits from the parent element like with other elements.

bdo Element - <bdo>

Element represents explicit text directionality formatting control for its children, BDO stands for "Bi-Directional Override". It allows authors to override the Unicode bidirectional algorithm, by explicitly specifying a direction override.

bdo Attributes - Authors must specify the global dir attribute, with the value ltr to specify left-to-right override, and rtl to specify right-to-left override; the auto value must not be specified.
dir - (Required)
Specifies the direction of text inside the <bdo> element: <bdo dir="value">

em Element - <em>

Element defines emphasized text, typically displayed in italic; placement of stress emphasis changes meaning of the sentence, thus it forms an integral part of the content. The level of stress a piece of content has is given by its number of ancestor <em> elements; way in which stress is used in this way depends on the language. A screen reader will pronounce the words in emphasis, using verbal stress.

i Element - <i>

Element defines a part of text in an alternate voice or mood, the content inside is typically displayed in italic. The tag is often used to indicate a technical term, phrase from another language, thought, name, etc. Terms in languages different from the main text, should be annotated with the global lang attribute.

strong Element - <strong>

Element represents strong importance, seriousness or urgency for its contents; content inside is typically displayed in bold. The relative level of importance of content is given by its number of ancestor <strong> elements; each <strong> element increases importance of its contents.

u Element - <u>

Element defines span of text articulated and styled differently from normal text, such as misspelled words or proper names; content inside is typically displayed with an underline. The default rendering in visual presentations clashes with conventional rendering of hyperlinks, as both have underlining. Authors are encouraged to avoid using the <u> element, where it could be confused for a hyperlink.

Line Breaking & Whitespace Elements

br Element - <br>

Element defines a line break (a new line) without starting a new paragraph, it is an empty element, which means it has no end tag. The <br> element is useful when writing addresses or poems.

pre element - <pre>

Element represents a block of preformatted text, in which structure is represented by typographic conventions rather than by elements. The text is displayed in a fixed-width font (usually Courier), and preserves both spaces and line breaks; it is displayed as written in the HTML source code. Authors are encouraged to consider how preformatted text will be experienced, as will be the case for users of speech synthesizers, braille displays and the like.

wbr Element - <wbr>

Element defines a possible line-break, a 'Word Break Opportunity', the tag specifies where in a text it would be ok to add a line-break. When a word is too long, the browser might break it at the wrong place; you can use the <wbr> element to add word break opportunities.

Quotation and Citation

abbr Element - <abbr>

Element defines an abbreviation or an acronym, optionally with its expansion like "HTML", "CSS", "Mr.", "Dr.", "ASAP", "ATM". The title attribute may be used to provide an expansion (a description) of the abbreviation, when you mouse over the element. Marking abbreviations can give useful information to browsers, translation systems and search-engines.

address Element - <address>

Element defines contact information for the author/owner of a document or article, and usually renders in italic; browsers will add a line break before and after. Element represents contact information for its nearest <article> or <body> element ancestor. The contact information can be an email address, URL, physical address, phone number, social media handle, etc.

blockquote Element - <blockquote>

Element defines a section that is quoted from another source, a browsers will usually indent <blockquote> elements.

blockquote Attributes - Content inside a blockquote quoted from another source, may be cited in the cite attribute.
cite
Specifies the source of the quotation, a link to the source of the quotation or more information about the edit: <blockquote cite="value">

cite Element - <cite>

Element defines the title of a work, and the element will usually render in italic. It can be work quoted/referenced in detail (a citation), or it can be a work mentioned in passing.

dfn Element - <dfn>

Element specifies the defining instance of a term that is going to be defined within the content. The paragraph, description list group or section that is the nearest ancestor of the <dfn> element, also contains the definition(s) for the term given by the element. If the title attribute is present, it must contain only the term being defined; the exact value of that attribute is the term being defined. The term inside the <dfn> tag can be with an <abbr> tag, or with the id attribute added. Then, whenever a term is used, it can refer back to the definition with a link.[]

figure Element - <figure>

Element specifies self-contained content (flow content), optionally with a caption and typically referenced as a single unit from the main flow of the document. The <figcaption> element child of the element, if any, represents the caption of the <figure> element's contents. When a figure is referred to from the main content by identifying it by its caption (e.g., figure number), it enables such content to be easily moved away from that primary content. The element can thus be used to annotate illustrations, diagrams, photos, code listings, etc.

figcaption Element - <figcaption>

Element represents a caption or legend for the contents of the element's parent <figure> element, and can contain additional information about the source. The <figcaption> element, can be placed as the first or last child of the <figure> element.

mark Element - <mark>

Element defines run of text, marked or highlighted for reference purpose due to its relevance in another context. When used in the main prose of a document, it indicates part of the document that has been highlighted due to its likely relevance to user's current activity.

q Element - <q>

Element defines a short quotation or phrasing content from another source, browsers normally insert quotation marks around the quotation - Use the <blockquote> element for long quotations. Quotation punctuation, quoting contents of the element, must not appear immediately before, after or inside of <q> elements; they will be inserted into the rendering by the user agent.

q Attributes - Content inside a <q> element quoted from another source, may be cited in the cite attribute.
cite
Specifies the source of the quote, a link to the source of the quotation or more information about the edit: <q cite="value">

ruby Element - <ruby>

Element defines a ruby annotation, it allows one or more spans of phrasing content to be marked with ruby annotations. Ruby annotations are short runs of text presented alongside base text, and is primarily used in East Asian typography, as a guide for pronunciation or to include other annotations. This kind of annotation is often used in Japanese publications, in Japanese, this form of typography is also known as furigana. Use ruby together with <rt> and <rp> elements, the ruby element consists of one or more characters that needs an explanation/pronunciation. An <rt> element gives that information, and an optional <rp> tag, defines what to show for browsers that do not support ruby annotations. The <ruby> element represents the union of the segments of base text it contains, along with the mapping from those base text segments to annotation segments. The <ruby> and <rt> elements can be used for a variety of kinds of annotations, including in particular those described here - {the ruby element}.

rt Element - <rt>

Element defines an explanation/pronunciation of characters for East Asian typography. The element marks the ruby text component of a ruby annotation, when it is the child of a <ruby> element; it doesn't represent anything itself. Use the <rt> tag together with <ruby> and <rp> elements, the <ruby> element consists of one or more characters that needs an explanation/pronunciation. And an <rt> element gives that information, an optional <rp> element defines what to show for browsers that do not support ruby annotations.

rp Element - <rp>

Element can be used to provide parentheses or other content around a ruby text component of a ruby annotation, to be shown by user agents that don't support ruby annotations. Use the <rp> tag together with <ruby> and <rt> elements, the <ruby> element consists of one or more characters that needs an explanation/pronunciation. An <rt> element gives that information, and the optional <rp> element defines what to show for browsers that do not support ruby annotations.

small Element - <small>

Element defines smaller text, represents side comments such as small print; it is only intended for short runs of text. Small print typically features disclaimers, caveats, legal restrictions or copyrights; it is also sometimes used for attribution, or licensing requirements.

sub Element - <sub>

Element defines subscripted text, the text appears half a character below the normal line, and is sometimes rendered in a smaller font. It can be used for chemical formulas (H2O), and mathematical expressions also often use subscripts and superscripts - Use the <sup> tag to define superscripted text.

sup Element - <sup>

Element defines superscripted text, the appears half a character above the normal line, and is sometimes rendered in a smaller font. In certain languages, superscripts are part of typographical conventions for abbreviations. Mathematical expressions also often use subscripts and superscripts - Use the <sub> tag to define subscript text.

<acronym>

Element defines an acronym, the tag was used in HTML 4 to define an acronym - Not supported in HTML5, use <abbr> instead.

Computer Code Elements

HTML contains several elements for defining user input and computer code.

code Element - <code>

Element represents a fragment of computer code, the content inside is displayed in the browser's default monospace font.

kbd Element - <kbd>

Element defines user input, although it may also be used to represent other input (eg: voice commands); content inside is displayed in the browser's default monospace font. When nested inside another <kbd> element, it represents an actual key, or other single unit of input as appropriate for the input mechanism.

var Element - <var>

Element represents a variable, and could be used in a mathematical or programming context, a constant, symbol, function parameter or term used as a placeholder in prose. The content inside the <var> element, is typically displayed in italic.

samp Element - <samp>

Element represents sample/quoted output from another program or computing system, the content is displayed in the browser's default monospace font.

Document Edits

del Element - <del>

Element defines text that has been deleted (a removal) from a document, browsers usually strike a line through deleted text.

del Attributes - The cite attribute may be used to specify a document, that explains the change; they are primarily intended for private use, and not readers.
cite
The source of the quotation or more information about the edit, that explains reason why the text was inserted/changed: <del cite="value">
datetime
May be used to specify the date, and optionally a time, when the text was inserted or changed. If present, the datetime attribute's value, must be a valid date string with an optional time: <del datetime="YYYY-MM-DDThh:mm:ssTZD">

ins Element - <ins>

Element defines text that has been inserted into a document (an addition), browsers usually underline inserted text.

ins Attributes
cite
The source of the quotation, or more information about the edit; this explains the reason why the text was inserted or changed: <ins cite="value">
datetime
May be used to specify the date, and optionally a time, of when the text was inserted or changed. If present, the datetime attribute's value must be a valid date string with an optional time: <ins datetime="YYYY-MM-DDThh:mm:ssTZD">

s Element - <s>

Element defines text that is no longer correct, accurate or relevant; the text will be displayed with a line through it in the browser's default monospace font. Is not appropriate when indicating document edits, to mark a span of text as having been removed from a document, use the <del> element.

<strike>

Element defines strikethrough text - Not supported in HTML5, use <del> or <s> instead.

Lists

Lists allow web developers to group a set of related items in lists, the lists can also be nested (a list inside of a list).

li Element - <li>

Element represents a list item, if its parent element is an <ol>, <ul> or <menu> element, then the element is an item of the parent element's list, as defined for those elements. Otherwise, the item has no defined list-related relationship to any other <li> element.

li Attributes
value
Is used to determine the ordinal value (start value) of the list item, when the <li>'s tag list owner is an <ol> element. The value attribute, if present, must be a valid integer; list items will increment from that number: <li value="value">

Unordered HTML List

An unordered list starts with the <ul> tag, and each list item starts with the <li> tag. The list items will be marked with bullets (small black circles) by default.

ul Element - <ul>

Element defines an unordered list, the items will be marked with bullets (small black circles) by default; items are the <li> element child nodes of the element.

menu Element - <menu>

Element is a semantic alternative to the <ul> element, to express an unordered list of commands, as a "toolbar"; list items will usually be displayed with bullet points. Each of which represents a command the user can perform or activate, it is used for context menus, toolbars and listing form controls or commands.

menu Attributes
label
Specifies a visible label for the menu, this attribute is often used to label nested menus (a menu within a menu): <menu label="value">
type
Specifies which type of menu to display: <menu type="list|context|toolbar"> { list - (Default) Specifies a list menu, a list of commands (<li> elements). | context - Specifies a context menu, the menu must be activated before user can interact with commands. | toolbar - Specifies a toolbar menu, active commands, allows user to interact with commands immediately. }

<dir>

Element defines a directory list - Not supported in HTML5, use <ul> instead.

Ordered HTML List

An ordered list starts with the <ol> tag, and each list item starts with the <li> tag; the list items will be marked with numbers by default.

ol Element - <ol>

Element defines an ordered list, where items are intentionally ordered as numerical or alphabetical; the items are <li> element child nodes of the <ol> element.

ol Attributes
reversed
The reversed attribute is a boolean attribute, if present, it indicates that the list is a descending list (order should be reversed: 3, 2, 1): <ol reversed>
start
The start attribute, if present, must be a valid integer; it is used to determine the starting value of the list. This value is always an integer, even when the numbering type is letters or roman; by default, an ordered list will start counting from 1: <ol start"value">
type
The type attribute can be used to specify the kind of marker to use in the list, in cases where that matters (e.g. items are to be referenced by their number/letter). If specified, it must have a value that is identical to one of the characters given: <ol type="1|a|A|i|I"> { 1 - Decimal numbers. | a - Lowercase latin alphabet. | A - Uppercase latin alphabet. | i - Lowercase roman numerals. | I - Uppercase roman numerals. }

HTML Description Lists

HTML also supports description lists, a description list is a list of terms, with a description of each term. The <dl> tag defines the description list, the <dt> tag defines the term (name) and the <dd> tag describes each term.

dl Element - <dl>

Element defines a description/association list, a description list is a list of terms with a description (name-value) for each. A name-value group consists of one or more names (<dt> elements), followed by one or more values (<dd> elements). Within a single <dl> element, there should not be more than one <dt> element for each name. Name-value groups may be terms and definitions, metadata topics and values, questions and answers, or any other groups of name-value data.

dt Element - <dt>

Element defines the term or name of a description group in a description list (<dl> element). The tag is used in conjunction with the <dl> tag, witch defines a description list, and <dd> witch describes each term/name.

dd Element - <dd>

Element represents the description, definition or value, part of a term-description or name group in a description list (<dl> element). The tag is used in conjunction with <dl>, witch defines a description list and <dt> witch defines terms/names. Inside a <dd> tag, you can put paragraphs, line breaks, images, links, lists, etc.

Cascading Style Sheets (CSS)

CSS stands for Cascading Style Sheets, it's the language used to style HTML and describes how HTML elements should be displayed. With CSS you can control color, font, text size, spacing, elements position, background images or colors, and much more. The word cascading means a style applied to a parent element, will also apply to all children elements within the parent. If color of the body text is set to "blue", all headings, paragraphs and other text elements within the body, will also get the same color (unless specify something else). CSS can save a lot of work, you can control the layout of multiple web pages at once, including the layout and variations for different devices and screen sizes.
CSS is added to HTML documents in 3 ways:

  1. Inline - An inline CSS is used to apply a unique style to a single HTML element, and uses the global style attribute on an HTML element: <p style="color:red;">A red paragraph.</p> - A red paragraph. - The property is a CSS property, and the value is a CSS value.
  2. Internal - An internal CSS is used to define a style for a single HTML page, using the <style> element, in the webpage's <head> section: <style> .css-types {list-style-type:upper-roman; background-color:lightblue} </style> - This internal style is what changes our list type (using the class attribute 'css-types') to ‘upper-roman’ characters, and gives it a ‘lightblue’ background.
  3. External - To use an external style sheet, add a <link> Element in the <head> section of each HTML page; to link to the external CSS file(s). The most common way to add CSS is to keep the styles in external CSS files, the external style sheet can be written in any text editor. The file must not contain any HTML code, and must be saved with a .css extension: <link rel="stylesheet" href="stylesheet.css"> - External style sheets can also be referenced with a full URL or with a path relative to the current web page. With an external style sheet, you can change the look of an entire web site, by changing one file.
Some commonly used CSS properties
Property Definition
color Defines the text color to be used.
font-family Defines the font to be used.
font-size Defines the text size to be used.
border Defines a border around an HTML element.
padding Defines a padding (space) between the text and the border.
margin Defines a margin (space) outside the border.

The id and class Attributes - The global id attribute is used to point to a specific style declaration, and the global class attribute is often used to point to a class name in a style sheet.

  • Syntax for id: Write a hash character (#), followed by an id name; then define the CSS properties within curly braces {}: #ElementId { property1: value; property2: value; }
  • Syntax for class: Write a period character (.), followed by a class name; then define the CSS properties within curly braces {}: .className { property1: value; property2: value; }

style Element - <style>

Element is used to define style information (CSS) for a document, inside of the <style> element you specify how HTML elements should render. It allows authors to embed CSS style sheets into their documents, the element does not represent any content for the user. When a browser reads a style sheet, it will format the HTML document according to the information in the style sheet; to link to an external style sheet, use the <link> tag. If properties have been defined for the same 'selector' (element), in different style sheets; the value from the last read style sheet will be used.

style Attributes - The title attribute on style elements defines CSS style sheet sets. The title attribute on style elements, like <link>, will differ from the global title attribute; in that a style block without a title, does not inherit the title of the parent element, it merely has no title.
media
The media attribute says which media/device the styles apply to or are optimized for, the value must be a valid media query list. This attribute is used to specify that the style is for special devices (like iPhone), speech or print media: <style media="value" - The default, if this attribute omitted, is "all"; meaning by default styles apply to all media.
    Devices:
  • all - (Default) Suitable for all devices.
  • aural - Speech synthesizers.
  • braille - Braille feedback devices.
  • handheld - Handheld devices (small screen and limited bandwidth).
  • projection - Projectors.
  • print - Print preview mode/printed pages.
  • screen - Computer screens.
  • tty - Teletypes and similar media using a fixed-pitch character grid.
  • tv - Television type devices (low resolution and limited scroll ability).
    Possible Operators:
  • and - Specifies an AND operator.
  • not - Specifies a NOT operator.
  • , - Specifies an OR operator.
    Values:
  • width - Specifies the width of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (min-width:500px)"
  • height - Specifies the height of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (max-height:700px)"
  • device-width - Specifies the width of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (device-width:500px)"
  • device-height - Specifies the height of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (device-height:500px)"
  • orientation - Specifies the orientation of the target display/paper, possible values are "portrait" or "landscape": media="all and (orientation: landscape)"
  • aspect-ratio - Specifies the width/height ratio of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (aspect-ratio:16/9)"
  • device-aspect-ratio - Specifies the device-width/device-height ratio of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (aspect-ratio:16/9)"
  • color - Specifies the bits per color of target display, "min-" and "max-" prefixes can be used: media="screen and (color:3)"
  • color-index - Specifies the number of colors the target display can handle, "min-" and "max-" prefixes can be used: media="screen and (min-color-index:256)"
  • monochrome - Specifies the bits per pixel in a monochrome frame buffer, "min-" and "max-" prefixes can be used: media="screen and (monochrome:2)"
  • resolution - Specifies the pixel density (dpi or dpcm) of the target display/paper, "min-" and "max-" prefixes can be used: media="print and (resolution:300dpi)"
  • scan - Specifies scanning method of a tv display, possible values are "progressive" and "interlace": media="tv and (scan:interlace)"
  • grid - Specifies if the output device is grid or bitmap, possible values are "1" for grid, and "0" otherwise: media="handheld and (grid:1)"
type
The type attribute specifies the Internet media type (formerly known as MIME type) of the style tag or style sheet. For now, the only supported value and the default value, is "text/css", which indicates that the content is CSS: <style type="value"> - Look at IANA Media Types , for a complete list of standard media types.

HTML Colors

HTML colors are specified with predefined color names and HEX, RGB, RGBA, HSL or HSLA values:

  • Color Names - In HTML, a color can be specified by using a color name, HTML supports 140 standard color names; all modern browsers support them - See the Complete List.
  • Hexadecimal Color - In HTML, a hexadecimal color is specified with: #RRGGBB - The hexadecimal integers, RR for red, GG for green and BB for blue specify the components of the color. Where RR, GG and BB are hexadecimal values between 00 and ff (same as decimal 0-255).
  • RGB - An RGB color value represents Red, Green, and Blue light sources: rgb(red, green, blue) - Each parameter for red, green and blue, defines the intensity of the color; with a value between 0 and 255.
  • RGBA - RGBA color values are an extension of RGB color values with an Alpha channel, which specifies the opacity for a color: rgba(red, green, blue, alpha) - The alpha parameter is a number between 0.0 (fully transparent) and 1.0 (not transparent at all).
  • HSL - HSL stands for hue, saturation and lightness, in HTML, a color can be specified in the form: hsl(hue, saturation, lightness) - Hue is a degree on the color wheel from 0 to 360, 0 is red, 120 is green, and 240 is blue. Saturation is a percentage value, 0% means a shade of gray and 100% is the full color; saturation can be described as the intensity of a color. Lightness is also a percentage value, 0% is black and 100% is white; the lightness of a color can be described as how much light to give the color.
  • HSLA - HSLA color values are an extension of HSL color values, with an Alpha channel, which specifies the opacity for a color: hsla(hue, saturation, lightness, alpha) - The alpha parameter is a number between 0.0 (fully transparent) and 1.0 (not transparent at all).

Note: HTML was never intended to contain tags for formatting a web page, it was created to describe the content (using elements) of a web page. When font and color attributes were added to HTML 3.2, it started a nightmare for web developers; information was added to every page, a long and expensive process. To solve this, the World Wide Web Consortium (W3C) created CSS; CSS removed the style formatting from the HTML page.
