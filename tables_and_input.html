Tables

HTML tables allow web developers to arrange data into rows, columns and cells inside of a <table> element; the rows and columns form a grid. Each table row starts with a new <tr> element, "tr" stands for table row. You can also add a caption that serves as a heading for the entire table, to add a caption to a table, use the <caption> tag. The <caption> tag should be inserted immediately after the <table> tag.

    An HTML table has two kinds of cells:
  • Header cells - Contains header information, and is created with a <th> element; "th" stands for table header. Sometimes you want cells to be table header cells, by default, the text in <th> elements are bold and centered (can change that with CSS). To use the first column as table headers, define the first cell in each row with a <th> element.
  • Data cells - Contains data created with a <td> element, "td" stands for table data; the text in <td> elements are regular and left-aligned by default. The table data elements are data containers of the table, they can contain all sorts of HTML elements; text, images, lists, other tables etc.

HTML tables can also have cells that span over multiple rows and/or columns, and can use CSS properties such as: borders, color, size, padding, etc. In visual media, providing column and row borders, as well as alternating row backgrounds; can be very an effective way to make complicated tables more readable. The <colgroup> element is used to style specific columns of a table, each group is specified with a <col> element. There is a very limited selection of legal CSS properties for colgroups.
Any table in general, where the reader might have difficulty understanding content, authors should include explanatory information introducing the table. This information is useful for all users, but especially for users who cannot see the table (e.g. users of screen readers). Such explanatory information should introduce the purpose of the table, outline its cell structure, highlight trends or patterns and teach how to use the table.

table Element - <table>

Element represents data with more than one dimension, in the form of a table. Tables have rows, columns and cells given by their descendants, their rows and columns form a grid. An HTML table consists of one <table> element and one or more table row <tr>, table header <th> and table data <td> elements. Authors are encouraged to provide information describing how to interpret complex tables. You can add a caption that serves as a heading for the entire table, using the <caption> tag.

caption Element - <caption>

Element represents a title of the table (a table caption) that is its parent, if it has a parent <table> element; it must be inserted immediately after the <table> tag. A caption can introduce context for a table, making it significantly easier to understand; by default, a table caption will be center-aligned above a table.

tr Element - <tr>

Element defines a row of cells in a table, a <tr> element contains one or more <th> or <td> elements.

th element - <th>

Element defines a header cell in a table, elements are bold and centered by default.

th Attributes
colspan
Number of columns that the cell is to span: <th colspan="value"> - If the value is colspan="0", this tells the browser to span the cell to the last column of the column group (colgroup).
rowspan
Number of rows that the cell is to span: <th rowspan="value"> - If the value is rowspan="0", this tells the browser to span the cell to the last row of the table section (thead, tbody or tfoot).
headers
The headers attribute specifies a space-separated list of id's, to one or more header cells the header cell is related to: <headers="value"> - The headers attribute has no visual effect in ordinary web browsers, but can be used by screen readers.
scope
The scope attribute is an enumerated attribute with five states, four of which have explicit keywords. This specifies whether a header cell is a header for a column, row, or group of columns and rows. The scope attribute has no visual effect in ordinary web browsers, but can be used by screen readers: <scope="auto|col|row|colgroup|rowgroup"> { auto - (Default) The scope attribute's missing value default and invalid value default are the auto state. The auto state makes the header cell apply to a set of cells selected based on context. | col - The col keyword, maps to the column state, the column state means the header cell applies to some of the subsequent cells in the same column(s). | row - The row state means the header cell applies to some of the subsequent cells in the same row(s). | colgroup - The colgroup keyword, maps to the column group state, the column group state means the header cell applies to all remaining cells in the column group. A th element's scope attribute must not be in the column group state if the element is not anchored in a column group. | rowgroup - The rowgroup keyword, maps to the row group state, the row group state means the header cell applies to all remaining cells in the row group. A <th> element's scope attribute must not be in the row group state if the element is not anchored in a row group. }
abbr
The abbr attribute specifies an abbreviated version of the content in a header cell, when referencing the cell in other contexts. It is typically an abbreviated form of the full header cell, but can also be an expansion, or merely a different phrasing: <th abbr="value"> - The abbr attribute has no visual effect in ordinary web browsers, but can be used by screen readers.

td Element - <td>

Element defines a data cell in a table, and can contain all sorts of HTML elements, such as: text, images, lists, other tables, etc. The text in <td> elements are regular and left-aligned by default.

td Attributes
colspan
The colspan attribute defines the number of columns a cell should span: <colspan="value"> - If the value is colspan="0", this tells the browser to span the cell to the last column of the column group (colgroup) - Only Firefox supports colspan="0".
rowspan
The rowspan attribute specifies the number of rows a cell should span: <td rowspan="value"> - If the value is, rowspan="0", this tells the browser to span the cell to the last row of the table section (thead, tbody, or tfoot) - Chrome, Firefox and Opera 12 (and earlier versions) support rowspan="0".
headers
The headers attribute specifies a space-separated list of id's, to one or more header cells the table cell is related to: <td headers="value"> - The headers attribute has no visual effect in ordinary web browsers, but can be used by screen readers.

thead Element - <thead>

Element represents a block of rows that consist of the column labels (it groups the column headers) for the parent <table> element, if has a parent and it is a table. The element is used in conjunction with the <tbody> and <tfoot> elements, to specify each part of a table (header, body, footer). Browsers can use these elements, to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page. The <thead> element must have one or more <tr> tags inside, and be used in the following context: As a child of a <table> element, after any <caption> or <colgroup> elements, and before any <tbody>, <tfoot> and <tr> elements.

tbody Element - <tbody>

Element represents a block of rows, that consist of a body of data for the parent <table> element (it groups the body content in a table), if it has a parent and it is a table. The element is used in conjunction with the <thead> and <tfoot> elements, to specify each part of a table (body, header, footer). Browsers can use these elements, to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page. The <tbody> element must have one or more <tr> tags inside, and be used in the following context: As a child of a <table> element, after any <caption>, <colgroup> and <thead> elements.

tfoot Element - <tfoot>

Element represents block of rows that consist of the column summaries (it groups the footer content) for parent <table> element, if has a parent and is a table. The <tfoot> element is used in conjunction with the <thead> and <tbody> elements, to specify each part of a table (footer, header, body). Browsers can use these elements, to enable scrolling of the table body independently of the header and footer. Also, when printing a large table that spans multiple pages, these elements can enable the table header and footer to be printed at the top and bottom of each page. The <tfoot> element must have one or more <tr> tags inside, and must be used in the following context: As a child of a <table> element, after any <caption>, <colgroup>, <thead> and <tbody> elements.

colgroup Element - <colgroup>

Element represents a group of one or more columns in the table (for formatting) that is its parent, if it has a parent and that is a <table> element. The <colgroup> tag must be a child of a <table> element, after any <caption> elements and before any <thead>, <tbody>, <tfoot> and <tr> elements. The tag is useful for applying styles to entire columns, instead of repeating the styles for each cell or each row. To define different properties to a column within a colgroup, use the <col> tag within the <colgroup> element: <colgroup> <col span="2" style="property:value"> </colgroup> - If you want to style more columns with different styles, use more <col> elements inside the <colgroup> tag. If a colgroup contains no <col> elements, then may have a span attribute specified, whose value must be a non-negative integer greater than zero and less than or equal to 1000.

colgroup Attributes
span
The span attribute defines the number of columns a <colgroup> element should span: <colgroup span="value">

col Element - <col>

Element specifies column properties for each column within a <colgroup> element. If the element has a parent <colgroup> element, with a parent <table> element, the <col> element represents one or more columns in the column group.

col Attributes - Element may have a span attribute specified, whose value must be a valid non-negative integer greater than zero and less than or equal to 1000.
span
The span attribute defines the number of columns a <col> element should span: <col span="value">

HTML Forms

An HTML form is used to collect user input, most often this is sent to a server for processing. A form is a component of a web page that has form controls, such as text, buttons, checkboxes, range or color picker controls. Writing a form consists of several steps, which can be performed in any order: writing the user interface, implementing server-side processing and configuring user interface to communicate with the server. Any form starts with a <form> element, inside which are placed the controls; the <form> element can contain one or more of the following form elements: <input>, <textarea>, <button>, <select>, <option>, <optgroup>, <fieldset>, <label or <output> - Many of these form controls have attributes to control exactly what values can be specified.

Most controls are represented by the <input> element, which by default provides a text control; the <input> element is the most used form element. An <input> element can be displayed in many ways, depending on the type attribute. And each input field must have a name attribute to be submitted, if the attribute is omitted, the value of the input field will not be sent at all. Multiple controls can have the same name, the server distinguishes for example, which checkbox was checked by seeing which values are submitted with that name. To label a control, the <label> element is used; the label text and the control itself go inside of the <label> element. The <label> element is useful for screen-reader users, because the screen-reader will read out loud the label when the user focuses on the input element. To group a batch of controls together, such as, radio buttons, one can use the <fieldset> element. The title of such a group of controls is given by the first element in the fieldset, which has to be a <legend> element.

form element - <form>

Element defines an HTML form for user input, and is a container for different types of form-associated elements, some represent editable values that can be submitted to a server.

form Attributes - The action, enctype, method, novalidate and target attributes are attributes for form submission.
accept-charset
The accept-charset attribute gives the character encodings to be used for the submission, as a space-separated list of one or more character encodings. If specified, value must be an ASCII case-insensitive match for "UTF-8": <form accept-charset="value>
In theory, any character encoding can be used, but no browser understands all of them. The more widely a character encoding is used, the better the chance that a browser will understand it.
    Common values:
  • UTF-8 - Character encoding for Unicode.
  • ISO-8859-1 - Character encoding for the Latin alphabet.
To view all available character encodings, see the Character Sets Reference.
action
The action attribute specifies where to send the form-data when a form is submitted: <form action="value">
    Possible values:
  • Absolute URL - Points to another web site: action="http://www.website.com/webpage.htm"
  • Relative URL - Points to a file within a web site: action="localpage.htm"
autocomplete
The autocomplete attribute specifies whether a form should have autocomplete on or off, the attribute is an enumerated attribute with two states: <form autocomplete="on|off"> { on - (Default) The browser will automatically complete values based on values that the user has entered before. The missing value default and the invalid value default are the on state. | off - The browser does not automatically complete entries, and the user must enter a value into each field for every use. }
enctype
The enctype attribute specifies how the form-data should be encoded when submitting it to the server, as an entry list. The enctype attribute can be used only if method="post": <form enctype=application/x-www-form-urlencoded|multipart/form-data|text/plain> { application/x-www-form-urlencoded - (Default) All characters are encoded before sent, spaces are converted to "+" symbols and special characters converted to ASCII HEX values. | multipart/form-data - This value is necessary if the user will upload a file through the form. | text/plain - Sends data without any encoding at all - Not recommended. }
method
The method attribute specifies how to send form-data (HTTP method to use), the form-data is sent to the page specified in the action attribute. The form-data can be sent as URL variables with get or as HTTP post transaction with post: <form method=get|post> { get - (Default) Appends the form-data to the URL in name/value pairs, the length of a URL is limited to about 3000 characters: URL?name=value&name=value - Never use get to send sensitive data, as it will be visible in the URL, get is better for non-secure data, like query strings in Google or form submissions where user wants to bookmark the result. | post - Sends the form-data as an HTTP post transaction, appends form-data inside the body of the HTTP request; the data is not shown in URL. Has no size limitations, and form submissions with post cannot be bookmarked (unlike the get method). }
name
The name attribute represents the form's name within the forms collection (specifies the name of a form). The value must not be the empty string, and must be unique amongst form elements in the forms collection that it is in (if any). The name attribute is used to reference elements in a JavaScript, or to reference form data after a form is submitted: <form name="value">
novalidate
The novalidate attribute is a boolean attribute, it specifies the form-data (input) should not be validated when submitted (bypass form control validation): <form novalidate>
target
The target attribute specifies a name or keyword that indicates where to display the response that is received after submitting the form (e.g. tab, window or inline frame): <form target="_self|_blank|_parent|_top|framename"> { _self - (Default) The response is displayed in the same frame. | _blank - The response is displayed in a new window or tab. | _parent - The response is displayed in the parent frame. | _top - The response is displayed in the full body of the window. | framename - The response is displayed in a named iframe. }
rel
The rel attribute specifies the relationship between the current document and the linked resource, controls what kinds of links the elements create: <form rel="external|help|license|next|nofollow|noopener|noreferrer|opener|prev|search> { external - Specifies that the referenced document is not a part of the current site. | help - Links to a help document. | license - Links to copyright information for the document. | next - The next document in a selection. | nofollow - Links to an unendorsed document, like a paid link; is used by Google, to specify the Google search spider should not follow that link. | noopener [] | noreferrer - Specifies that the browser should not send a HTTP referrer header if the user follows the hyperlink. | opener [] | prev - The previous document in a selection. | search - Links to a search tool for the document. }

input Element - <input>

Element defines an input control (input field), where the user can enter data; it is the most important/used form element. The input element represents a "typed" data field, usually with a form control to allow the user to edit the data. But it can also be displayed in several ways depending on the type attribute, witch controls the data type (and associated control) of the element; it is an enumerated attribute.

input Attributes
accept
The accept attribute specifies a filter for what file types the user can pick from the file input dialog box (only for type="file"). To specify more than one value, separate the values with a comma: <input accept="audio/*,video/*,image/*"/> - Do not use this attribute as a validation tool, file uploads should be validated on the server: <input accept="file_extension|audio/*|video/*|image/*|media_type"> { file_extension - Specify the file extension(s) (e.g: .gif, .jpg, .png, .doc) the user can pick from. | audio/* - The user can pick all sound files. | video/* - The user can pick all video files. | image/* - The user can pick all image files. | media_type - A valid media type, with no parameters; look at IANA Media Types , for a complete list of standard media types. }
alt
The alt attribute provides an alternate text for the user, if they cannot view the image; specifies an alternate text for images (only for type="image"): <input alt="value">
autocomplete
The autocomplete attribute specifies whether or not an input field should have autocomplete enabled. When a user starts to type in a field, the browser should display options to fill in the field, based on earlier typed values. The autocomplete attribute works with the following input types: text, search, url, tel, email, password, datepickers, range and color: <input autocomplete=on|off> { on - (Default) Specifies that autocomplete is on (enabled). | off - Specifies that autocomplete is off (disabled). }
autofocus
The autofocus attribute is a boolean attribute, it specifies that an input element should automatically get focus when the page loads: <input autofocus>
checked
The checked content attribute is a boolean attribute that gives the default checkedness of the input element. It specifies that an <input> element should be pre-selected when the page loads for type="checkbox" or type="radio": <inputchecked>
dirname
The dirname attribute enables the submission of the text direction of the input field; specifies text direction will be submitted: <input name="inputname" dirname=inputname.dir> - The dirname attribute's value is always the name of the input field, followed by ".dir".
disabled
The disabled attribute is a boolean attribute, used to make the control non-interactive and to prevent its value from being submitted. When present, it specifies that the input element should be disabled, a disabled input element is unusable and un-clickable. Disabled input elements in a form will not be submitted: <input disabled>
form
The form attribute is used to explicitly associate the input element with its form owner, it specifies the form an input element belongs to. The value of this attribute must be the id attribute of a <form> element in the same document: <input form="value">
formaction
The formaction attribute specifies the URL of the file that will process the input control when the form is submitted. The formaction attribute overrides the action attribute of the <form> element, and is used with type="submit" and type="image": <input formaction="value"> { - Possible values: Absolute URL - The full address of a page, like: href="http://www.website.com/aresult.asp" | Relative URL - Points to a file within the current site, like: href="localresult.asp" }
formenctype
The formenctype attribute specifies how form-data should be encoded when submitting it to the server, as an entry list encoding type; only for forms with method="post". The formenctype attribute is used with type="submit" and type="image", attribute overrides the enctype attribute of the <form> element: <input formenctype=application/x-www-form-urlencoded|multipart/form-data|text/plain> { application/x-www-form-urlencoded - (Default) All characters are encoded before sent, spaces are converted to "+" symbols, and special characters are converted to ASCII HEX values. | multipart/form-data - This value is necessary if the user will upload a file through the form. | text/plain - Sends data without any encoding at all - Not recommended. }
formmethod
The formmethod attribute defines the HTTP method for sending form-data to the action URL, can be sent as URL variables "get" or as an HTTP post transaction "post". The formmethod attribute can be used with type="submit" and type="image", and it overrides the method attribute of the <form> element: <input formmethod="get|post"> { get - (Default) Appends the form-data to the URL in name/value pairs: URL?name=value&name=value. There is a limit to how much data can place in a URL and it varies between browsers, therefore, cannot be sure all of form-data will be correctly transferred. Never use the "get" method to pass sensitive information, password or other sensitive information will be visible in the browser's address bar. This method is useful for form submissions where a user want to bookmark the result. | post - Sends the form-data as an HTTP post transaction, method is more robust and secure than "get", and does not have size limitations. Form submissions with the "post" method cannot be bookmarked like with the "get" method. }
formnovalidate
The formnovalidate attribute overrides the novalidate attribute of the <form> element, defines that form elements should not be validated when submitted. The formnovalidate attribute is a boolean attribute, and can be used with type="submit"; it can be set in the following ways: <input formnovalidate> { <input formnovalidate="formnovalidate"> | <input formnovalidate=""> }
formtarget
The formtarget attribute specifies a name or a keyword that indicates where to display the response that is received after submitting the form. The formtarget attribute can be used with type="submit" and type="image", and it overrides the target attribute of the <form> element: <input formtarget="_self|_blank|_parent|_top|framename"> { _self - (Default) The response is displayed in the same frame. | _blank - The response is displayed in a new window or tab. | _parent - The response is displayed in the parent frame. | _top - The response is displayed in the full body of the window. | framename - The response is displayed in a named iframe. }
height
The height attribute specifies the height of the input element or vertical dimension, and is only for type="image"; the height is in pixels (height="100"). - Always specify both height and width attributes for images, if set, the space required for the image is reserved when the page is loaded. However, without these attributes, the browser does not know the size of the image, and cannot reserve the appropriate space to it: <input height="value">
list
The list attribute is used to identify an element that lists predefined options (autocomplete options) suggested to the user. It refers to a <datalist> element that contains pre-defined options for an <input> element, its value must be the ID of a datalist element in the same tree: <input list="value">
max
The max attribute specifies the maximum value for an input element: <input max="number|date"> { number - Specifies the maximum value allowed. | date - Specifies the maximum date allowed. }
Use the max attribute together with the min attribute to create a range of legal values, the attributes work with the following input types: number, range, date, datetime-local, month, time and week.
maxlength
The maxlength attribute specifies the maximum number of characters allowed in the input element, the default value is 524288: <input maxlength="value">
min
The min attribute specifies the minimum value for an input element: <input min="number|date"> { number - Specifies the minimum value allowed. | date - Specifies the minimum date allowed. }
Use the min attribute together with the max attribute to create a range of legal values, the attributes work with the following input types: number, range, date, datetime-local, month, time and week.
minlength
The minlength attribute specifies the minimum number of characters required in an input field: <input minlength="value"> - The minlength attribute can be used with input type: text, search, url, tel, email and password.
multiple
The multiple attribute is a boolean attribute that indicates whether the user is to be allowed to specify more than one value: <input multiple> - The multiple attribute works with the following input types: email and file. To select multiple files, hold down the CTRL or SHIFT key while selecting. For type="email", separate each email with a comma: mail@example.com, mail2@example.com, mail3@example.com in the email field.
name
The name attribute represents the element's name, to use for form submission; only form elements with a name attribute will have their values passed when submitting a form: <input name="value">
pattern
The pattern attribute specifies a regular expression against which the control's value are to be checked: <input pattern="value"> - If specified, the attribute's value must match the JavaScript Pattern [+UnicodeMode, +N] production; learn more about regular expressions, in the JavaScript tutorial.
When an input element has a pattern attribute specified, authors should also include a title attribute to give a description of the pattern. Additional information could also be included, so long it assists the user in filling in the control; otherwise, assistive technology would be impaired. User agents may use contents of this attribute, when informing the user the pattern is not matched. Or at any other suitable time, such as in a tooltip or read out by assistive technology when the control gains focus. You cannot submit this form when the field is incorrect, and the pattern attribute works with the following input types: text, date, search, url, tel, email and password.
placeholder
The placeholder attribute represents a short hint (a word or short phrase) intended to aid the user with data entry when the control has no value: <input placeholder"value">
The short hint is displayed in the input field before the user enters a value, and works with the following input types: text, search, url, tel, email, and password. The placeholder attribute should not be used as an alternative to a label, for a longer hint or advisory text, the title attribute is more appropriate.
popovertarget
The popovertarget attribute value must be the ID of an element with a popover attribute in the same tree as the input or button with the popovertarget attribute.[]
popovertargetaction
The popovertargetaction attribute is an enumerated attribute: <button popovertarget="id" popovertargetaction="toggle|show|hide"> { toggle - Shows or hides the targeted popover element, the popovertargetaction attribute's invalid value default and missing value default are both the toggle state. | show - Shows the targeted popover element. | hide - Hides the targeted popover element. } - Whenever possible ensure the popover element is placed immediately after its triggering element in the DOM. Doing so will help ensure the popover is exposed in a logical programmatic reading order for users of assistive technology.
readonly
The readonly attribute is a boolean attribute that controls whether or not the user can edit the form control (is read-only). A read-only input field cannot be modified, a user can tab to it, highlight and copy text from it; a form will still submit an input field that is readonly: <input readonly>
required
The required attribute is a boolean attribute, specifies an input field must be filled out before submitting the form (is required): <input required> - The required attribute works with the following input types: text, search, url, tel, email, password, date pickers, number, checkbox, radio and file.
size
The size attribute gives the number of characters in a visual rendering, the user agent is to allow the user to see while editing the element's value. The size attribute, if specified, must have a value that is a valid non-negative integer greater than zero; the default value is 20: <input size="value"> - The size attribute works with the following input types: text, search, tel, url, email and password.
src
The src attribute specifies the URL of the image to use as a submit button, attribute can only be used with, and is required for type="image": <input src="value"> { - Possible values: Absolute URL -Points to another web site, like: src="http://www.website.com/submit.gif" | Relative URL - Points to a file within a web site, like: src="localsubmit.gif" }
step
The step attribute indicates the granularity that is expected (and required) of the value or values, by limiting the allowed values; the default is 1: <input step="value">
For Example, if step="3", the legal numbers could be -3, 0, 3, 6, etc. If specified, must either have a value that is a valid floating-point number that parses to a number greater than zero. Or must have a value that is an ASCII case-insensitive match for the string "any". The step attribute can be used together with the max and min attributes to create a range of legal values. The step attribute works with the following input types: number, range, date, datetime-local, month, time and week.
type
The type attribute controls the data type and associated control of the element, it is an enumerated attribute. If the type attribute is not specified, the default type is "text"; always use the <label> tag to define labels for input: <input type="button|checkbox|color|date|datetime-local|email|file|hidden|image|month|number|password|radio|range|reset|search|submit|tel|text|time|url|week"
  • text - (Default) The input element represents a one line plain text edit control for the element's value (a text field); default width of the text field is 20 characters: <input type="text"> - The value attribute, if specified, must have a value that contains no Line Feed (LF) or Carriage Return (CR) characters.
  • button - The input element represents a clickable button with no default behavior, is mostly used with a JavaScript to activate a script. A label for the button must be provided in the value attribute, though it may be the empty string: <input type="button">
  • checkbox - The input element represents a two-state control that represents the element's checkedness state, a checkbox is shown as a square box that is ticked (checked) when activated. If the element's checkedness state is true, the control represents a positive selection and if it is false, a negative selection. Checkboxes are used to let a user select one or more options of a limited number of choices, always add the <label> tag for best accessibility practices: <input type="checkbox">
  • color - The input element represents a color well control (a color picker), for setting the element's value to a string representing a simple color; the default value is #000000 (black). The value attribute, if specified and not empty, must have a value that is a valid simple color; it must be in seven-character hexadecimal notation. Always add the <label> tag for best accessibility practices: <input type="color">
  • date - The input element represents a control for setting the element's value to a string representing a specific date (a date picker), resulting value includes the year, month and day. The value attribute, if specified and not empty, must have a value that is a valid date string. The step attribute is expressed in days, the default step is 1 day; and min or max attributes, must also have a value that is a valid date string. Always add the <label> tag for best accessibility practices: <input type="date">
  • datetime-local - The input element represents a control for setting the element's value to a string representing a local date and time, with no time-zone offset information (a date picker - year, month, day and time). The value attribute, if specified and not empty, must have a value that is a valid local date and time string. Element represents a caption in a user interface, the caption can be associated with a specific form control. textarea Element - <textarea>

Element defines a multiline input control (a text area), it is often used in a form to collect user inputs, like comments or reviews; the contents of the control represent it's default value. A text area can hold an unlimited number of characters, and the text renders in a fixed-width font (usually Courier). The id attribute is needed to associate the text area with a label, always add the <label> tag for best accessibility practices.

textarea Attributes - The size of a text area is specified by the cols and rows attributes, or using CSS.
autocomplete
The autocomplete attribute controls how the user agent provides autofill behavior.[]
autofocus
The autofocus attribute is a boolean attribute, it specifies that the text area should automatically get focus when the page loads: <textarea autofocus>
cols
The cols attribute specifies the expected maximum number of characters per line (the visible width of a text area); the default value is 20. If the cols attribute is specified, its value must be a valid non-negative integer greater than zero: <textarea cols="value"> - The size of a text area can also be set by the CSS height and width properties.
dirname
The dirname attribute controls how the element's directionality is submitted, as the name of a form control to use. The dirname attribute's value is always the name of the text area, followed by ".dir": <textarea name="textareaname" dirname="textareaname.dir">
disabled
The disabled attribute is a boolean attribute, when present, it specifies that the text area should be disabled. Is used to make the control non-interactive and prevent its value from being submitted; a disabled text area is unusable and the text is not selectable (cannot be copied): <textarea disabled>
form
The form attribute is used to explicitly associate the textarea element with its form owner. The value of this attribute must be equal to the id attribute of a <form> element in the same document: <textarea form=form_id>
maxlength
The maxlength attribute specifies the maximum number of characters (maximum length in characters) allowed in the text area: <textarea maxlength="value">
minlength
The minlength attribute is a form control minlength attribute, it specifies the minimum length of value: <textarea minlength="value">
name
The name attribute specifies a name for a text area, is needed to reference the form data after the form is submitted. Is used to reference elements in a JavaScript, or reference form data after a form is submitted; if omit, no data from the text area will be submitted: <textarea name="value">
placeholder
The placeholder attribute represents a short hint (a word or short phrase) intended to aid the user with data entry when the control has no value. A hint could be a sample value or a brief description of the expected format, the short hint is displayed in the text area before the user enters a value: <textarea placeholder="value"> - For a longer hint or advisory text, the title attribute is more appropriate, the placeholder attribute should also not be used as an alternative to a label.
readonly
The readonly attribute is a boolean attribute used to control whether the text can be edited by the user or not (read-only). In a read-only text area, the content cannot be changed, but a user can tab to it or highlight and copy content from it: <textarea "value">
required
The required attribute is a boolean attribute, when specified, the user will be required to enter a value before submitting the form: <textarea required>
rows
The rows attribute specifies the visible height of a text area, in lines; the default value is 2. If the rows attribute is specified, its value must be a valid non-negative integer greater than zero: <textarea rows="value"> - The size of a textarea can also be specified by the CSS height and width properties.
wrap
The wrap attribute specifies how the text in a text area is to be wrapped when submitted in a form, is an enumerated attribute with two keywords and states: <textarea wrap="hard|soft"> { soft - (Default) The soft keyword maps to the Soft state, and indicates text in the textarea is not to be wrapped when submitted; though can still be wrapped in the rendering. | hard - The hard keyword maps to the Hard state, and indicates text in the textarea is to have newlines added (by user agent), so text is wrapped when submitted. If the element's wrap attribute is in the Hard state, the cols attribute must be specified. }

button Element - <button>

Element represents a clickable button labeled by its contents, inside you can put text and tags; not possible with a button created with an <input> element.

button Attributes - Always specify the type attribute, to tell browsers what type of button it is; it controls the behavior of the button when it is activated. The formaction, formenctype, formmethod, formnovalidate and formtarget attributes are attributes for form submission; they must not be specified if the type attribute is not in the Submit Button state.
autofocus
The autofocus attribute is a boolean attribute, when present, it specifies that a button should automatically get focus when the page loads: <button type="button" autofocus>
disabled
The disabled attribute is a boolean attribute, when present, it specifies that the button should be disabled. Is used to make the control non-interactive and to prevent its value from being submitted; button is unusable and un-clickable: <button disabled>
form
The form attribute is used to explicitly associate the button element with its form owner. The value of this attribute must be equal to the id attribute of a <form> element in the same document: <button form="value">
formaction
The formaction attribute specifies where to send the form-data when a form is submitted, this attribute overrides the form's action attribute. The formaction attribute is only used for buttons with type="submit": <button type="submit" formaction="value"> { Possible values: Absolute URL - The full address of a page, like: href="http://www.website.com/formresult.asp" | Relative URL - Points to a file within the current site, like: href="localresult.asp" }
formenctype
The formenctype attribute specifies how form-data should be encoded (entry list encoding type) before sending it to a server, attribute overrides the form's enctype attribute. The formenctype attribute is only used for buttons with type="submit": <button type="submit" formenctype="application/x-www-form-urlencoded|multipart/form-data|text/plain"> { application/x-www-form-urlencoded - (Default) All characters will be encoded before sent. | multipart/form-data - This value is necessary if the user will upload a file through the form. | text/plain - Sends data without any encoding at all - Not recommended. }
formmethod
The formmethod attribute specifies which HTTP method to use when sending the form-data, attribute overrides the form's method attribute. The formmethod attribute is only used for buttons with type="submit": <button type="submit" formmethod="get|post"> { get - It appends the form-data to the URL in name/value pairs: URL?name=value&name=value. Is a limit to how much data can place in a URL (varies between browsers), therefore, cannot be sure all the form-data will be correctly transferred. Never use the "get" method to pass sensitive information, password or other sensitive information will be visible in the browser's address bar. It is useful for form submissions where a user want to bookmark the result. | post - It sends the form-data as an HTTP post transaction, it does not have size limitations; and is more robust and secure than "get". Form submissions with the "post" method cannot be bookmarked, as with the "get" method. }
formnovalidate
The formnovalidate attribute is a boolean attribute, it specifies the form-data should not be validated on submission; attribute overrides form's novalidate attribute. The formnovalidate attribute can be used to make submit buttons that do not trigger the constraint validation, the attribute is only used for buttons with type="submit": <button type="submit" formnovalidate>
formtarget
The formtarget attribute specifies where to display the response after submitting the form, attribute overrides the form's target attribute. The formtarget attribute is only used for buttons with type="submit": <button type="submit" formtarget="_self|_blank|_parent|_top|framename"> { _self - (Default) Loads the response in the same frame. | _blank - Loads the response in a new window/tab. | _parent - Loads the response in the parent frame. | _top - Loads the response in the full body of the window. | framename - Loads the response in a named iframe. }
name
The name attribute specifies the name for a <button> element, is used to reference form-data after the form has been submitted, or reference element in a JavaScript. Several <button> elements can share the same name, this allows to have buttons with equal names, which can submit different values when used in a form: <button name="value">
popovertarget
The popovertarget attribute value must be the ID of an element with a popover attribute in the same tree as the input or button with the popovertarget attribute.[]
popovertargetaction
The popovertargetaction attribute is an enumerated attribute: <button popovertarget="id" popovertargetaction="toggle|show|hide"> { toggle - Shows or hides the targeted popover element, the popovertargetaction attribute's invalid value default and missing value default are both the toggle state. | show - Shows the targeted popover element. | hide - Hides the targeted popover element. } - Whenever possible ensure the popover element is placed immediately after its triggering element in the DOM. Doing so will help ensure the popover is exposed in a logical programmatic reading order for users of assistive technology.
type
The type attribute specifies the type of button, it is an enumerated attribute; that controls the behavior of the button when it is activated. Always specify the type attribute for the <button> element, different browsers may use different default types: <button type="button|reset|submit"> { submit - (Default) The button is a submit button and submits form-data. | button - The button is a clickable button. | reset - The button is a reset button and resets the form-data to its initial values. }
value
The value attribute gives the element's value (initial value of the button) for the purposes of form submission. The element's value is the value of the element's value attribute, if there is one, or the empty string otherwise. In a form, the button and its value is only submitted if the button itself was used to submit the form: <button value="value"> - If use a <button> element in an HTML form, Internet Explorer (prior to version 8), will submit the text between the tags. While the other browsers will submit the content of the value attribute.

search Element - <search>

Element represents a part of a document or application that contains a set of form controls, or other content related to performing a search or filtering operation. This could be a search of the web site or application, a way of searching or filtering search results on the current web page, or a global or Internet-wide search function. meter Element - <meter>

Element defines a scalar measurement within a known range or a fractional value (such as disk usage), also known as a 'gauge'. Authors are encouraged to include a textual representation of the gauge's state in the element's contents, for user agents that do not support the <meter> element. The element should not be used to indicate progress (as in a progress bar), for that role, HTML provides a separate <progress> element. The meter element also does not represent a scalar value of arbitrary range, it would be wrong to use this to report weight or height, unless is a known maximum value. Always add the <label> tag for best accessibility practices.

meter Attributes - The value attribute must be specified, when used with microdata (machine-readable data), the value attribute provides the element's machine-readable value. User agents combine the value of the title attribute and the other attributes to provide context-sensitive help or inline text detailing the actual values.
value (Required)
The value attribute must be specified, and must have a value that is a valid floating-point number. The value attribute specifies the value to have the gauge indicate as the "measured" or current value of the element. The value attribute must be between the min and max attribute values: <meter value="value"> - If the min attribute specified, then let minimum be that attribute's value; otherwise, let it be zero. If the max attribute is specified, then let maximum be that attribute's value; otherwise, let it be 1.0.
min
The min attribute specifies the lower bound of the range, and must have a value that is a valid floating-point number; default value is 0. The min attribute, together with the max attribute, specifies the full range of the gauge; the value must be less than the max attribute value: <meter min=number>
max
The max attribute specifies the upper bound or maximum value of the range, and must have a value that is a valid floating-point number; default value is "1". The max attribute, together with the min attribute, specifies the full range of the gauge; the value must be greater than the min attribute value: <meter max="value">
low
The low attribute specifies the range considered to be the "low" part/value or High limit of low range; and must have a value that is a valid floating-point number. The low attribute value must be greater than the min attribute value, and it also must be less than the high and max attribute values: <meter low="value">
high
The high attribute specifies the range considered to be the "high" part/value or Low limit of high range; and must have a value that is a valid floating-point number. The high attribute value must be less than the max attribute value, and it also must be greater than the low and min attribute values: <meter high="value">
optimum
The optimum attribute gives the position that is "optimum"; and must have values that are valid floating-point numbers: <meter optimum="value"> - If that is higher than the "high" value then this indicates that the higher the value, the better. If it's lower than the "low" mark then it indicates that lower values are better. If it is in between then it indicates that neither high nor low values are good.
form
The form attribute specifies the form the <meter> tag belongs to, the value must be equal to the id attribute of a <form> or <meter> element in the document: <meter form="value">

select Element - <select>

Element defines a drop-down list, and is most often used in a form to collect user input; it represents a control for selecting amongst a set of options. The <option> tags inside the <select> element, define the available options in the drop-down list. The list of options for a <select> element, consists of all the <option> element children of a <select> element. And all of the <option> element children, of all <optgroup> element children of the <select> element (in tree order). Always add the <label> tag for best accessibility practices, the id attribute is needed to associate the drop-down list with a label.

select Attributes - The name attribute is needed to reference the form data after the form is submitted, if omitted, no data from the drop-down list will be submitted.
autocomplete
Hint for form autofill feature, the attribute controls how the user agent provides autofill behavior.[]
disabled
The disabled attribute is a boolean attribute, it specifies that the drop-down list should be disabled. Is used to make the control non-interactive and to prevent its value from being submitted; a disabled drop-down list is unusable and un-clickable. A select element that is not disabled is mutable: <select disabled>
form
The form attribute is used to explicitly associate the select element with its form owner, it specifies the form the drop-down list belongs to. The value of this attribute must be equal to the id attribute of a <form> element in the same document: <select form="value">
multiple
The multiple attribute is a boolean attribute, if present, the select element represents a control for selecting zero or more options from the list of options. If the attribute absent, the select element represents a control for selecting a single option from the list of options: <select multiple> { Selecting multiple options vary in different operating systems and browsers: For windows - Hold down the control (ctrl) button to select multiple options. | For Mac - Hold down the command button to select multiple options. } - Because of the different ways of doing this, and because you have to inform user multiple selection is available, is more user-friendly to use checkboxes instead.
name
The name attribute specifies the name for a drop-down list: <select name="value"> - The name attribute is used to reference elements in a JavaScript, or to reference form data after a form is submitted.
required
The required attribute is a boolean attribute, when present, it specifies the user is required to select a value before submitting the form: <select required>
size
The size attribute gives the number of options to show to the user in a drop-down list. The size attribute, must have a value that is a valid non-negative integer greater than zero; the default value is 1, and if the multiple attribute present, default value is 4. If the value greater than 1, but lower than total number of options in the list, the browser will add a scroll bar to indicate there are more options: <select size=number> - In Chrome and Safari, this attribute may not work as expected for size="2" and size="3".
autofocus
The autofocus attribute is a boolean attribute, it specifies that the drop-down list should automatically get focus when the page loads: <select autofocus>

option Element - <option>

Element represents an option in a <select> element, or as part of a list of suggestions in a <datalist> element; by default the first item in the drop-down list is selected. The <option> element(s) go inside of a <select>, <optgroup> or <datalist> element; if have a long list of options, group related options within the <optgroup> tag.

option Attributes - The element can be used without any attributes, but usually needs the value attribute; which indicates what is sent to the server on form submission.
disabled
The disabled attribute is a boolean attribute, it specifies that an option should be disabled; a disabled option is unusable and un-clickable. An option element is disabled if its disabled attribute is present, or if it is a child of an optgroup element whose disabled attribute is present: <option disabled>
label
The label attribute provides a label for element, the label of an option element is the value of the label content attribute. The attribute specifies a shorter version of an option; the shorter version will be displayed in the drop-down list: <option label "value">
selected
The selected attribute is a boolean attribute, it specifies an option should be pre-selected when the page loads (selected by default). The pre-selected option will be displayed first in the drop-down list, the selected attribute can also be set after the page loads, with a JavaScript. A select element whose multiple attribute is not specified, must not have more than one descendant option element with its selected attribute set: <option selected>
value
The value attribute provides a value for element, and specifies the value to be sent to a server (value used for form submission). If the value attribute is not specified, the content will be passed as a value instead: <option value="value">

optgroup Element - <optgroup>

Element represents a group of <option> elements with a common label, it is used to group related options in a <select> element (a drop-down list). Element's group of <option> elements consists of the <option> elements, that are children of the <optgroup> element. If you have a long list of options, groups of related options are easier to handle for a user.

optgroup Attributes - The label attribute must be specified.
disabled
The disabled attribute is a boolean attribute and can be used to disable a group of option (option-group) elements together; a disabled option-group is unusable and un-clickable: <optgroup disabled>
label
The label attribute must be specified, the attribute specifies a label/description for an option-group: < optgroup label="value">

datalist Element - <datalist>

Element represents a set of <option> elements, that represent predefined options for other controls (input controls). Element is hooked up to input using the list attribute on the <input> element, users see a drop-down list of pre-defined options as they input data. The <datalist> tag is used to provide an "autocomplete" feature for <input> elements. Element can be used in two ways, the simplest case, the <datalist> element has just <option> element children. The more elaborate, the element can be given contents to be displayed for down-level clients that don't support the <datalist> tag. This case, <option> elements are provided inside a <select> element inside of the <datalist> element. Each <option> element that is a descendant of the <datalist> element, is not disabled, and whose value is a string that is empty; represents a suggestion, each has a value and label. The <datalist> element's id attribute must be equal to the <input> element's list attributel; this will bind them together.

fieldset Element - <fieldset>

Element represents a set of form controls or other content, grouped together, and will draw a box around the related elements; optionally with a caption. The caption is given by the first <legend> element that is a child of the <fieldset> element, the remainder of descendants form the group.

fieldset Attributes
disabled
The disabled attribute is a boolean attribute, it specifies that a group of related form elements (a fieldset) should be disabled. Excluding those that are descendants of the <fieldset> element's first legend element child (except any inside legend), if any. A disabled fieldset is unusable and un-clickable, a fieldset element is a disabled fieldset if it matches the following conditions: Its disabled attribute is specified, or is a descendant of another fieldset element whose disabled attribute is specified, and is not a descendant of element's first legend child: <fieldset disabled>
form
The form attribute is used to explicitly associate the fieldset element with its form owner (which form the fieldset belongs to). The value of this attribute must be equal to the id attribute of a <form> element in the same document: <fieldset form="value">

The name attribute specifies a name for a fieldset, is used to reference elements in a JavaScript, or to reference form data after a form is submitted: <fieldset name="value">

Element represents a caption for the rest of the contents of the <legend> element's parent <fieldset> element.

output Element - <output>

Element is used to represent the result of a calculation performed by the application, or result of a user action (eg: one performed by a script). This element can be contrasted with the <samp> element, which is the appropriate element for quoting output of other programs run previously.

output Attributes
for
The for attribute specifies the relationship between the result of a calculation, and the elements used in the calculation. Allows an explicit relationship to be made between a result, and the elements that went into or that influenced the calculation. The attribute, must contain a string of an unordered unique space-separated list of ids of one or more elements. None of which that are identical to another token, and each of which must have the value of an ID of an element in the same tree: <output for="value">
form
The form attribute is used to explicitly associate the output element with its form owner (specifies form the tag belongs to). The value of the form attribute must be equal to the id attribute of a <form> element in the same document: <output form="value">
name
The name attribute specifies the name of an <output> element, and is used to reference form data after it has been submitted, or to reference the element in a JavaScript. The <output> element is associated with a form so it can be easily referenced from the event handlers of form controls, value itself is not submitted when the form submitted: <output name="value">

Configuring a form to communicate with a server

Form submissions are exposed to servers in a variety of ways, most commonly as HTTP GET or POST requests. To specify the exact method used, the method attribute is specified on the <form> element. This doesn't specify how form data is encoded, though; to specify that, use the enctype attribute. Also, you have to specify the URL of the service that will handle the submitted data, using the action attribute. For each form control you want submitted, you then have to give a name that will be used to refer to the data in the submission. When a form is submitted, the data in the form is converted into the structure specified by the enctype attribute. Then it sent to the destination, specified by the action attribute, using the given method.

Client-side form validation

Forms can be annotated in such a way that the user agent will check the user's input before the form is submitted. The server still has to verify the input is valid (since hostile users can easily bypass the form validation) , but it allows the user to avoid the wait incurred by having the server be the sole checker of the user's input. The simplest annotation is the required attribute, which can be specified on <input> elements to indicate that the form is not to be submitted until a value is given. - When a form is submitted, invalid events are fired at each form control that is invalid. This can be useful for displaying a summary of the problems with the form, since typically the browser itself will only report one problem at a time.

Enabling client-side automatic filling of form controls

Some browsers attempt to aid the user by automatically filling form controls, rather than having the user reenter their information each time. For example, a field asking for the user's telephone number, can be automatically filled with the user's phone number. To help the user agent with this, the autocomplete attribute can be used to describe the field's purpose.

Improving the user experience on mobile devices

Some devices, in particular those with virtual keyboards can provide the user with multiple input modalities. For example, when typing in a credit card number, the user may wish to only see keys for digits 0-9; while when typing in their name, they may wish to see a form field that by default capitalizes each word. Using the inputmode attribute we can select appropriate input modalities.
