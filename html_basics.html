HTML Elements

Element Tags are used to delimit the start and end of Elements in the markup: <Start-Tag> Element's Content! </End-Tag> - Start and end tags, in certain cases, can be omitted and are implied by other tags. Some HTML elements will display correctly, even if you forget the end tag; however, never rely on this, as unexpected results and errors may occur - Never skip the End Tag. Elements whose content model is nothing are also, for convenience Void Elements or Empty Elements and are Elements that have no end tag: <Empty Element> .
HTML tags are not case sensitive, the HTML standard does not require lowercase tags, but W3C recommends lowercase in HTML - In stricter document types like XHTML, it demands lowercase.

Nested Elements

HTML elements can be 'nested' (contain other elements), all HTML Documents consist of Nested HTML Elements. Tags have to be nested, such that elements are all completely within each other, without overlapping (such as in the Sample HTML on the site's Main page): <head> <title>Page Tile</title> </head>

HTML Element Attributes

All Elements can have Attributes which control how the elements work, they provide additional information about elements. Attributes are always specified in the start tag and usually come in name value pairs, separated by an "=" character: name="value"
An attribute value is a text string, except where otherwise specified, the value can remain unquoted if it doesn't contain whitespace, or the following characters: " ' ` = < >. Otherwise, it has to be quoted using either single or double quotes; double quotes around attribute values are the most common, but single quotes can also be used. When the attribute value itself contains double quotes, it is necessary to use single quotes, or vice versa. The HTML Standard also does not require lowercase attribute names, however, W3C recommends to always use lowercase attributes names and quotes with attribute values.

HTML Global Attributes

Global attributes are attributes that can be used with all HTML elements - Click to see the HTML Global Attributes.
accesskey
The accesskey attribute's value is used by the user agent as a guide for creating a keyboard shortcut that activates or focuses the element. Each element that can be activated or focused can be assigned a single key combination to activate it. The accesskey attribute value must be a single character letter or digit, none of which identical to another token: accesskey="value"
Using accesskeys is difficult because they may conflict with other key standards in the browser, the way of accessing the shortcut key element is varying in different browsers. Most browsers will use accesskeys only if pressed together with the Alt key. Adapting accesskeys to all international languages are difficult, the accesskey value may not be present on all keyboards. Because of these concerns, it is advised not to use accesskeys.
 
autocapitalize
Some methods of entering text, for example on mobile virtual keyboards and voice input, often assist users by capitalizing the first letter of sentences. The autocapitalize attribute allows authors to control such behavior: autocapitalize="value"
A virtual keyboard that implements autocapitalization might automatically switch to showing uppercase letters. Other types of input, for example voice input, may perform autocapitalization in a way that does not give users an option to intervene first. The autocapitalize attribute, as typically implemented, does not affect behavior when typing on a physical keyboard. For this reason, as well as the ability for users to override or edit the behavior in some cases, the attribute must not be relied on for any input validation.
The autocapitalization processing model is based on selecting among five autocapitalization hints, defined as follows:
  • default - The user agent and input method should make their own determination of whether or not to enable autocapitalization.
  • none - No autocapitalization should be applied (all letters should default to lowercase).
  • sentences | on - The first letter of each sentence should default to a capital letter; all other letters should default to lowercase.
  • words - The first letter of each word should default to a capital letter; all other letters should default to lowercase.
  • characters - All letters should default to uppercase.
The autocapitalize attribute never causes autocapitalization to be enabled for input elements whose type attribute is in one of the URL, Email or Password states.
 
autofocus
The autofocus content attribute allows to indicate an element is to be focused as soon as the page is loaded, allowing user to start typing without having to focus the element: <element autofocus>
 
class
The HTML class attribute specifies one or more classnames for an element, multiple HTML elements can share the same class: class="value"
The class name is case sensitive, and there are no additional restrictions on tokens authors can use, but encouraged to use values that describe the nature of the content. HTML elements can also belong to more than one class, to define multiple classes, separate the class names with a space: class="Name1 Name2 Next-Name"
 
contenteditable
The contenteditable attribute specifies whether content of an element is editable or not, and is an enumerated attribute whose keywords are true, plaintext-only and false: contenteditable="true|plaintext-only|false"
In addition, there is a fourth state, the inherit state, which is the missing value default and the invalid value default.
  • true - The true state indicates that the element is editable, an 'empty-string' and the true keyword map to the true state.
  • plaintext-only - The plaintext-only state indicates that only the element's raw text content is editable, but rich text formatting is disabled.
  • false - The false state indicates that the element is not editable, false keyword maps to the false state.
  • inherit - The inherit state indicates that the element is the same kind of editable (or not editable) as the parent element.
 
data-*
The data-* attributes is used to store custom data private to the page or application, it gives the ability to embed custom data attributes on all HTML elements. The stored/custom data can then be used in the page's JavaScript to create a more engaging user experience (without Ajax calls or server-side database queries). Custom attributes prefixed with "data-" will be completely ignored by the user agent.
The data-* attributes consist of two parts: data-name="value"
  1. attribute name - The attribute name should not contain any uppercase letters, and must be at least one character long after the prefix "data-".
  2. attribute value - The attribute value can be any string.
 
dir
The dir attribute specifies the text direction of the element's content, and is an enumerated attribute with the following keywords and states: dir="ltr||rtl|auto"
  • ltr - (Default) The ltr keyword, maps to the ltr state, the contents of the element are explicitly directionally isolated left-to-right text.
  • rtl - The rtl keyword, maps to the rtl state, the contents of the element are explicitly directionally isolated right-to-left text.
  • auto - The auto keyword, maps to the auto state, the contents of the element are explicitly directionally isolated text. The direction is to be determined programmatically using the contents of the element (only recommended if the text direction is unknown).
Authors are strongly encouraged to use the dir attribute to indicate text direction rather than using CSS, that way documents will continue to render correctly even in the absence of CSS (e.g. as interpreted by search engines).
 
draggable
The draggable attribute specifies whether an element is draggable or not, an enumerated attribute; it has three states, true, false and auto: draggable="auto|true|false"
  1. auto - (Default) The first state is auto; it has no keywords but it is the missing value default and the invalid value default. The auto state uses the default behavior of the user agent (the browser).
  2. true - The second state is true and it has the keyword true, the true state means the element is draggable.
  3. false - The third state is false and it has the keyword false, the false state means that the element is not draggable.
An element with a draggable attribute should also have a title attribute that names the element for the purpose of non-visual interactions. Links and images are draggable by default, the draggable attribute is often used in drag and drop operations (JavaScript API).
 
enterkeyhint
The enterkeyhint content attribute is an enumerated attribute, that specifies what action label (or icon) to present for the enter key on virtual keyboards. Allows authors to customize the presentation of the enter key in order to make it more helpful for users. User agents can support the enterkeyhint attribute on form controls, or in elements in an editing host (e.g., using contenteditable): enterkeyhint="enter|done|go|next|previous|search|send"
  • v - The user agent should present a cue for the operation 'enter', typically inserting a new line.
  • done - The user agent should present a cue for the operation 'done', typically meaning is nothing more to input and the input method editor (IME) will be closed.
  • go - The user agent should present a cue for the operation 'go', typically meaning to take the user to the target of the text they typed.
  • next - The user agent should present a cue for the operation 'next', typically taking the user to the next field that will accept text.
  • previous - The user agent should present a cue for the operation 'previous', typically taking the user to the previous field that will accept text.
  • search - The user agent should present a cue for the operation 'search', typically taking user to the results of searching for the text they have typed.
  • send - The user agent should present a cue for the operation 'send', typically delivering the text to its target.
 
hidden
When specified on an element, indicates the element is not yet, or no longer, directly relevant to the page's current state. Or that it is being used to declare content to be reused by other parts of the page as opposed to being directly accessed by the user. The hidden attribute can be used to keep a user from seeing an element until some other condition has been met (like selecting a checkbox). Then, a JavaScript could remove the hidden attribute, and make the element visible.
The hidden attribute is an enumerated attribute, following lists the states for this attribute: <element hidden> | <element hidden="hidden|until-found">
  • hidden - When an element has the hidden attribute in the hidden state, user agents should not render elements.
  • until-found - When an element has the hidden attribute in the hidden until found state, the element is hidden but content inside the element will be accessible to find-in-page and fragment navigation. When these features attempt to scroll to a target which is in the element's subtree, the user agent will remove the hidden attribute in order to reveal the content before scrolling to it.
Because this attribute is typically implemented using CSS, it's also possible to override it using CSS. Authors are encouraged to make sure their style sheets don't change the 'display' or 'content-visibility' properties of hidden until found elements (legacy user agents). The for attributes of label and output elements that are not themselves hidden must similarly not refer to elements that are hidden; such references would cause user confusion. Elements and scripts may, however, refer to elements that are hidden in other contexts.
 
id
The id attribute specifies its element's unique identifier (id), the value of the id attribute must be unique within the HTML document. Can not have more than one element with the same id in an HTML document: id="value"
The id name is case sensitive, must contain at least one character, and must not contain any space characters (ASCII whitespace). There are no other restrictions on what form an ID can take, IDs can consist of just digits, punctuation, or start with an underscore. The id attribute most used to point to a style in a style sheet in CSS, and by JavaScript (via the HTML DOM) to manipulate the element with the specific id.
 
inert
Inert nodes generally cannot be focused, and user agents do not expose the inert nodes to accessibility APIs or assistive technologies. The inert content cannot be interacted with by anyone, user agents may allow to override the restrictions on find-in-page and text selection: <element inert>
An inert subtree should not contain content or controls which are critical to aspects of the page which are not in the inert state. Authors should not specify elements as inert unless the content they represent are also visually obscured in some way. In most cases, authors should not specify the attribute on individual form controls; these instances, the disabled attribute probably more appropriate. However, for many other situations authors strongly encouraged to clearly mark what parts of their document are active and which are inert, to avoid user confusion.
 
inputmode
The inputmode content attribute is an enumerated attribute, that specifies what kind of input mechanism would be most helpful for users entering content. User agents can support the inputmode attribute on form controls, or in elements in an editing host: inputmode="none|text|tel|url|email|numeric|decimal|search"
  • none - The user agent should not display a virtual keyboard, this keyword useful for content that renders its own keyboard control.
  • text - The user agent should display a virtual keyboard capable of text input in the user's locale.
  • tel - The user agent should display a virtual keyboard capable of telephone number input, including keys for the digits 0 to 9, the "#" and "*" characters. In some locales, this can also include alphabetic mnemonic labels, in the US, the key "2" historically also labeled with the letters A, B and C.
  • url - The user agent should display a virtual keyboard capable of text input in the user's locale, for aiding in input of URLs. The "/" and "." characters, and quick input of strings commonly found in domain names such as "www." or ".com".
  • email - The user agent should display a virtual keyboard capable of text input in the user's locale, with keys for aiding in input of email addresses, such the "@" character and "." characters.
  • numeric - The user agent should display a virtual keyboard capable of numeric input, this keyword is useful for PIN entry.
  • decimal - The user agent should display a virtual keyboard capable of fractional numeric input, numeric keys and the format separator for the locale should be shown.
  • search - The user agent should display a virtual keyboard optimized for search.
 
is
A customized built-in element, defined with an extends option - These types of custom elements have a local name equal to the value passed in their extends option. Their defined name is used as the value of the is attribute, which therefore must be a valid custom element name []: is="value"
 
itemid
Elements with an itemscope and itemtype attribute may also have an itemid attribute specified, this is to give a global identifier for the item. So it can be related to other items on pages elsewhere on the web, must have a value that is a valid URL potentially surrounded by spaces. The global identifier of an item is the value of its element's itemid attribute: itemid="value"
The itemid attribute must not be specified on elements that do not have both an itemscope attribute and an itemtype attribute specified.
 
itemprop
Every HTML element may have an itemprop attribute specified, if doing so adds one or more properties to one or more items. The attribute if specified, must have a value that is an unordered set of unique space-separated tokens none of which are identical to another token: <element itemprop="name">value</element>
Must ensure property names contain no full stop characters (.), colon characters (:) and no ASCII whitespace. The rules disallow colon characters (:) in non-URL values because otherwise they could not be distinguished from URLs. Values with full stop characters (.) reserved for future extensions, whitespace is disallowed because otherwise values would be parsed as multiple tokens.
 
itemref
Elements with an itemscope attribute may have an itemref attribute specified to give a list of additional elements to crawl to find the name-value pairs of the item. If specified, must have a value that is an unordered set of unique space-separated tokens none of which are identical to another token and consisting of IDs of elements in the same tree: itemref="value"
The itemref attribute must not be specified on elements that do not have an itemscope attribute specified.
 
itemscope
Every HTML element may have an itemscope attribute specified, creates a new item, a group of name-value pairs: <element itemscope>
Elements with an itemscope attribute may have an itemtype attribute specified, to give the item types of the item. Elements with an itemscope attribute may have an itemref attribute specified, to give a list of additional elements to crawl to find the name-value pairs of the item. Elements with an itemscope and itemtype attribute may also have an itemid attribute specified, to give a global identifier for the item, so it can be related to other items on pages elsewhere on the web.
 
itemtype
Elements with an itemscope attribute may have an itemtype attribute specified, to give the item types of the item. If specified, must have a value that is an unordered set of unique space-separated tokens. None of which are identical to another token and each of which is a valid URL string that is an absolute URL: itemtype="value1 value2"
The item types of an item are the tokens obtained by splitting the element's itemtype attribute's value on whitespace. Elements with an itemscope and itemtype attribute may also have an itemid attribute specified, to give a global identifier for the item, so it can be related to other items on pages elsewhere on the web.
 
lang
The lang attribute specifies the primary language for the element's contents and for any of the element's attributes that contain text. Its value must be a valid BCP 47 language tag, or the empty string; setting to the empty string indicates the primary language is unknown: lang="value"
If these attributes are omitted from an element, then the language of this element is the same as the language of its parent element. Are encouraged to specify a document lang attribute, common examples are "en" for English, "es" for Spanish, "fr" for French, and so on. Country codes can also be added to the language code, the first two characters define language, and the last two characters define the country: <html lang="en-US"> - Specifies the HTML language as English, and United States as the country.
See the HTML Language Code Reference and the Reference for Country Codes.
 
nonce
A nonce content attribute represents a cryptographic nonce (a number used once), the value is text. Elements that have a nonce content attribute, ensure that the cryptographic nonce is only exposed to script: nonce="value"
Can be used by Content Security Policy to determine whether or not a given fetch will be allowed to proceed.
 
popover
When specified, the element won't be rendered until it becomes shown, at which point it will be rendered on top of other page content. The popover attribute is an enumerated attribute with the auto or manual keywords and states: popover="auto|manual"
  • auto - Closes other popovers when opened; has light dismiss.
  • manual - Does not close other popovers, does not light dismiss.
Light dismiss means that pressing Esc key or clicking outside of a popover whose popover attribute is in the auto state will close the popover. A popover can be useful for rendering a status message, confirming the action performed by the user. Authors should use the appropriate ARIA attributes to ensure the popover is accessible. Inserting a popover element into an output element will generally cause screen readers to announce the content when it becomes visible.
 
slot
The slot attribute is used to assign a slot to an element. An element with a slot attribute is assigned to the slot created by the slot element, whose name attribute's value matches that slot attribute's value. But only if that slot element finds itself in the shadow tree, whose root's host has the corresponding slot attribute value.[]
 
spellcheck
The spellcheck attribute specifies whether the element is to have its spelling and grammar checked or not, an enumerated attribute whose keywords are true and false: spellcheck="true|false|default"
  • true - The true keyword (or an empty string) map to the true state, indicates that the element is to have its spelling and grammar checked.
  • false - The false keyword maps to the false state, indicates that the element is not to be checked.
  • default - In addition, is a third state, the default state, indicates element to act according to a default behavior; possibly based on parent element's own spellcheck state.
Text values in input elements (not password), can be spellchecked; or text in textarea elements and in editable elements.
 
style
The style attribute specifies an inline style for an element, and is a style attribute as defined by the CSS Style Attributes. The property is a CSS property, the value is a CSS value, and it is used to add styles to an element; such as color, font, size and more. One or more CSS properties and values separated by semicolons: style="property:value;property:value;">
 
tabindex
The tabindex attribute specifies the tab order of an element, when the "tab" button is used for navigating. Allow or prevent them from being sequentially focusable, and determine their relative ordering for sequential focus navigation: tabindex="value">
The name "tab index" comes from common use of the Tab key to navigate through focusable elements, term "tabbing" refers to moving forward through sequentially focusable areas. The attribute if specified, must have a value that is a valid integer; the tabbing order of the element, 1 is first. Positive numbers specify the relative position of the element's focusable areas in sequential focus navigation order; negative numbers indicate the control is not sequentially focusable. Should use caution when using values other than 0 or −1 for their tabindex attributes as this is complicated to do correctly.
  • omitted or non-integer values - The user agent will decide whether the element is focusable, and if is, whether sequentially focusable, click focusable or both.
  • -1 or negative integer values - Causes element to be focusable, and indicates the author would prefer the element to be click focusable but not sequentially focusable. The user agent might ignore this preference for click and sequential focusability, for example; specific element types according to platform conventions or for keyboard-only users.
  • 0 - Causes element to be focusable, and indicates author would prefer element to be both click focusable and sequentially focusable. The user agent might ignore this preference for click and sequential focusability.
  • positive integer values - Behaves same as 0, but in addition creates a relative ordering within a tabindex-ordered focus navigation scope, so elements with higher value come later.
The tabindex attribute cannot be used to make an element non-focusable, can do that by disabling element, or making it inert.
 
title
The title attribute represents advisory information or extra information for the element, such as would be appropriate for a tooltip. The information is most often shown as a tooltip text when the mouse moves over the element: title="value"
On a link, could be the title or a description of the target resource, an image the image credit or a description of the image, a paragraph a footnote or commentary on the text. Relying on the title attribute is currently discouraged as many user agents do not expose the attribute in an accessible manner as required by specification. For example, requiring a pointing device, which excludes keyboard-only users and touch-only users. If this attribute is omitted from an element, it implies the title attribute of the nearest ancestor HTML element with a title attribute set; setting the attribute overrides this. If the title attribute's value contains line feed (LF) characters, the content is split into multiple lines. Setting the attribute to the empty string indicates the element has no advisory information.
 
translate
The translate attribute specifies whether content of an element should be translated when the page is localized, and is an enumerated attribute; the keywords are yes and no: translate="yes|no|inherit"
  • yes - The empty string and the yes keyword map to the yes state, specifies that the content of the element should be translated. When an element is in the translate-enabled state, the element's translatable attributes and the values of its Text node children are to be translated when the page is localized.
  • no - The no keyword maps to the no state, specifies that the content of the element must not be translated. When an element is in the no-translate state, the element's attribute values and values of its Text node children are to be left as-is when page is localized.
  • inherit - In addition, there is a third state, the inherit state, which is the missing value default and the invalid value default. The element's translation mode is in the same state as its parent element's, if any.

Accessible Rich Internet Applications (ARIA)

A screen reader is a software program that reads the HTML code, and allows the user to "listen" to the content. Screen readers are useful for people who are visually impaired or learning disabled. To enable assistive technology to expose a more fine-grained interface, a set of annotations for assistive technology products can be specified; the ARIA role and aria-* attributes. The role attribute is defined in Accessible Rich Internet Applications (ARIA); in addition with the aria-* content attributes.
Learn more about Web Accessibility at W3's Accessibility Introduction.

Main Elements

document Element - <!DOCTYPE>

HTML documents must start with <!DOCTYPE>, the declaration is not an HTML tag, it is information to the browser about what document type to expect. It must only appear once, at the top of the page (before any HTML tags); if omitted browsers tend to use a different rendering mode incompatible with some specifications. Every HTML document is represented by a Document Object; the Document object's origin is defined in DOM. The declaration not case sensitive, in HTML5, the declaration is: <!DOCTYPE html>

  • Each Document has a policy container, initially a new policy container, which contains policies which apply to the Document.
  • Each Document has a permissions policy, which is initially empty.
  • Each Document has a module map, which is initially empty.
  • Each Document has a cross-origin opener policy, which is initially a new cross-origin opener policy.
  • Each Document has a navigation id, which is initially null.
In older documents, the declaration is more complicated because the declaration must refer to a DTD (Document Type Definition): <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"> - HTML 4.01

html Element - <html>

The html element represents the root of an HTML document, the tag is the container for all other HTML elements (except <!DOCTYPE>); it is what the HTML document itself begins with. Authors are encouraged to specify a lang attribute on the root html element, giving the document's language. This is meant to assist search engines, browsers and aids speech synthesis or translation tools to determine what pronunciation or rules to use: <html lang="en"> - Declares the document's language is English.
See all the language codes in the HTML ISO Language Code Reference.

html Attributes
xmlns
Specifies the XML namespace attribute, if need content to conform to XHTML. The namespace "xmlns=http://www.w3.org/1999/xhtml" is default, and will be added to the html tag even if you do not include it.

head Element - <head>

The head element is used as the first element in an html element, and represents a container/collection of metadata for the Document (data about data). Metadata is data about the HTML document, it is not displayed and typically defines the document title, character set, styles, scripts and other meta information. []

title Element - <title>

The title element represents the document's title or name, the title must be text-only, and is shown in the browser's title bar or page's tab. There must be no more than one title element per document: <title>A Meaningful Page-Title</title>
Authors should use titles that identify their documents even when they are used out of context, try to make the title as accurate and meaningful as possible. If it's reasonable for the Document to have no title, then the title element is probably not required. The title element provides a title for the page when it is added to favorites and displays a title for the page in search engine-results. The content of a page title very important for search engine optimization (SEO), and is used by search engine algorithms to decide order when listing pages in search results.

body Element - <body>

The body element contains and represents all contents of an HTML document, the tag defines the visible part of the document's body. It contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc. There can only be one body element in an HTML document: See the Sample HTML example on the sites Main webpage.

HTML Comments

The comment tag is used to insert comments in the source code, HTML comments are not displayed in browsers; but can help document the HTML source code. You can add comments to HTML source code, by using the following syntax: <!-- This is an HTML comment --> - Notice there is an exclamation point (!) in the start tag, but not in the end tag. Place notifications and reminders in HTML code, or explain code, which can help to edit it later; this is especially useful if you have a lot of code. Comments can also be used to hide content, and they can hide parts in the middle of the HTML code. You can even hide more than one line, everything between the comment tag will be hidden from the display. Thus, comments are great for Debugging HTML, because you can comment out any HTML lines of code, one at a time, to search for errors.
