Embedded Content

Embedded content is content that imports another resource into the document, or content from another vocabulary inserted into the document. Some Embedded Content Elements can have 'Fallback Content', witch is content that is used when an external resource cannot be.

HTML Hyperlinks

Links are a conceptual construct, and represent a connection between two resources; they are created with: <a>, <area>, <form> and <link> elements. HTML Links are Hyperlinks, and are found in nearly all web pages; they allow users to click their way from page to page. A linked page is normally displayed in the current browser window, unless you specify another target. To indicate a resource be downloaded, rather than immediately used, the download attribute can be specified on <a> or <area> elements (creates the hyperlink to that resource). The HTML <a> tag defines a hyperlink, the most important attribute of this element is the href attribute; which indicates a link's destination. When you move the mouse over a link, the arrow will turn into a 'little hand'; by default, links will appear as follows in all browsers (Links can also be styled with CSS):

  • Unvisited Link - Is underlined and blue.
  • Visited Link - Is underlined and purple.
  • Active Link - Is underlined and red.
Interactive user agents may also provide users with a means to follow hyperlinks using the <link> element, somewhere within their user interface.

A file path describes the location of a file in a web site's folder structure, there are two kinds of links in HTML:

  1. Absolute URLs: Links to an External Resources, this uses a full URL to link to a web page; this links to an external image or document hosted on another website.
  2. Relative URLs: Witch are Hyperlinks, this is a local link to a page within the same website, and is specified with a relative URL; if the URL begins without a slash, it will be relative to the current page.

Bookmark Links

HTML links can be used to create bookmarks, so that readers can jump to specific parts of a web page; this is useful if a web page is very long. To create a bookmark, first create the bookmark, then add a link to it; when clicked, the page will scroll to the specified location of the bookmark:

  1. Use the id attribute to create a bookmark: <h2 id="C4">Chapter 4</h2>
  2. Then add a link to the bookmark, using the hash character (#) from within the same page: <a href="#C4">Jump to Chapter 4</a>
    Or, you can add a link to a bookmark on another page, by specifying the webpage URL: <a href="html_page.html#C4">Jump to Chapter 4 on the html_page</a>
To link to the top of the current page, you can use both of these methods: href="#top" | href="#"

A link does not have to only be text, it can also be an image or any other HTML element:

  • Image Links - To use an image as a link, just put an <img> tag inside of an <a> element's tag.
  • Email Address Links - Use mailto: inside of the href attribute, to create a link that opens the user's email program, and lets them send a new email: <a href="mailto:an_email@example.com">Send an E-Mail</a>
  • Button Links - To use an HTML button as a link, you have to add JavaScript code; this allows to specify what happens on certain events, like a button click.

nav Element - <nav>

Element represents a section of a page that links to other pages, or parts within the page; it is a section with navigation links. Not all groups of links on a page need to be in a <nav> element, it is primarily intended for sections that consist of major navigation blocks. It is common for footers to have a short list of links to various pages of a site, such as the terms of service, the home page or a copyright page. Screen readers, can also use this element to determine whether to omit the initial rendering of this content. You can use the <nav> element as a way to determine what content on the page to initially skip or provide on request (or both).

a Element - <a>

Element defines a Hyperlink and is used to link from one page to another, the link text is the part visible; clicking it will send the reader to the specified URL address.

a Attributes - The link's destination is specified in the href attribute, if specified, it represents a hypertext anchor labeled by its contents. If no href attribute is specified, then the element represents a placeholder for where a link might otherwise have been placed, consisting of just the element's contents. If the global itemprop attribute is specified, then the href attribute must also be specified. The target, download, ping, rel, hreflang, type and referrerpolicy attributes must be omitted if the href attribute is not present.
href
The href attribute specifies the URL of the page the link goes to (address of the hyperlink), and is the most important attribute of the <a> element: <a href="value"> { Possible values: Absolute URL - Points to another web site, like: href="http://www.website.com/an_html_page.htm" | Relative URL - Points to a file within a web site, like: href="local_.page.htm" | Element Id - Link to an element with a specified id within the page, like: href="#page-section" | Other Protocols - like an https://, ftp://, mailto:, file:, etc. | Script - Such as: href="javascript:alert('Hello');" }
target
The target attribute specifies where to open the linked document: <a target="_self|_blank|_parent|_top|framename"> { _self (Default) - Opens the linked document in the same frame as it was clicked. | _blank - Opens the linked document in a new window or tab. | _parent - Opens the linked document in the parent frame. | _top - Opens the linked document in the full body of the window. | framename - Opens the linked document in the named iframe. }
download
The download attribute specifies a target file specified in an href attribute, and will be downloaded when a user clicks on the hyperlink, instead of navigating to it. The optional value of the attribute will be the new name of the file after it is downloaded, if the value is omitted, the original filename is used. There are no restrictions on allowed values, and the browser will automatically detect the correct file extension, then add it to the file: <a download="value">
ping
The ping attribute specifies a list of URLs to be notified if the user follows the hyperlink (URLs to ping). When the user clicks on the hyperlink, the ping attribute will send a short HTTP POST request to the specified URL. Values must be a space separated list of one or more valid URLs, and is typically used for tracking: <a ping="value">
rel
The rel attribute specifies the relationship between the current document and the linked document, it is only used if the href attribute is present. Search engines can use this attribute to get more information about a link: <a rel=alternate|author|bookmark|external|help|license|next|nofollow|noreferrer|noopener|prev|search|tag> { alternate - Provides a link to an alternate representation of the document (i.e. print page, translated or mirror). | author - Provides a link to the author of the document. | bookmark - Permanent URL used for bookmarking. | external - Indicates that the referenced document is not part of the same site as the current document. | help - Provides a link to a help document. | license - Provides a link to licensing information for the document. | next - Provides a link to the next document in the series. | nofollow - Links to an unendorsed document, like a paid link; is used by Google, to specify the Google search spider should not follow that link. | noopener - Requires that any browsing context created by following the hyperlink must not have an opener browsing context. | noreferrer - Makes the referrer unknown, a no referer header will be included when the user clicks the hyperlink. | prev - The previous document in a selection. | search - Links to a search tool for the document. | tag - A tag (keyword) for the current document. }
hreflang
The hreflang attribute specifies the language of the linked document or resource, is purely advisory, and is only used if the href attribute is set. The two-letter language code specifies the language of the linked document, to view all available language codes, go to Language code reference: https://www.w3schools.com/tags/ref_language_codes.asp: <a hreflang="value">
type
The type attribute specifies the Internet media type (formerly known as MIME type) or hint of the linked document/resource. This attribute is purely advisory, and is only used if the href attribute is set: <a type="value"> - Look at IANA Media Types for a complete list of standard media types: http://www.iana.org/assignments/media-types/
referrerpolicy
The referrerpolicy attribute specifies which referrer information to send when the user clicks on the hyperlink (a referrer policy for fetches initiated by the element): <a referrerpolicy=no-referrer-when-downgrade|no-referrer|origin|origin-when-cross-origin|same-origin|strict-origin-when-cross-origin|unsafe-url> { no-referrer-when-downgrade (Default) - Sends the origin, path and query string, if the protocol security level stays the same, or is higher; sends nothing to a less secure level (HTTPS to HTTP is not ok). | no-referrer - No referrer information is sent. | origin - Sends the origin (scheme, host and port) of the document. | origin-when-cross-origin - Sends the origin of the document for cross-origin request, and sends the origin, path and query string for same-origin request. | same-origin - Sends a referrer for same-origin request, and sends no referrer for cross-origin request. | strict-origin-when-cross-origin - Sends the origin if the protocol security level stays the same or is higher; or sends nothing to a less secure level (like HTTPS to HTTP). | unsafe-url - Sends the origin, path and query string, regardless of security; use this value carefully. }
media
The media attribute specifies what media or device the linked document is optimized or designed for, like iPhone, speech or print media. This attribute is purely advisory, and only used if the href attribute is present; it can accept several values: a media="value">
    Devices:
  • all - (Default) Suitable for all devices.
  • aural - Speech synthesizers.
  • braille - Braille feedback devices.
  • handheld - Handheld devices (small screen and limited bandwidth).
  • projection - Projectors.
  • print - Print preview mode/printed pages.
  • screen - Computer screens.
  • tty - Teletypes and similar media using a fixed-pitch character grid.
  • tv - Television type devices (low resolution and limited scroll ability).
    Possible Operators:
  • and - Specifies an AND operator.
  • not - Specifies a NOT operator.
  • , - Specifies an OR operator.
    Values:
  • width - Specifies the width of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (min-width:500px)"
  • height - Specifies the height of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (max-height:700px)"
  • device-width - Specifies the width of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (device-width:500px)"
  • device-height - Specifies the height of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (device-height:500px)"
  • orientation - Specifies the orientation of the target display/paper, possible values are "portrait" or "landscape": media="all and (orientation: landscape)"
  • aspect-ratio - Specifies the width/height ratio of the targeted display area, "min-" and "max-" prefixes can be used: media="screen and (aspect-ratio:16/9)"
  • device-aspect-ratio - Specifies the device-width/device-height ratio of the target display/paper, "min-" and "max-" prefixes can be used: media="screen and (aspect-ratio:16/9)"
  • color - Specifies the bits per color of target display, "min-" and "max-" prefixes can be used: media="screen and (color:3)"
  • color-index - Specifies the number of colors the target display can handle, "min-" and "max-" prefixes can be used: media="screen and (min-color-index:256)"
  • monochrome - Specifies the bits per pixel in a monochrome frame buffer, "min-" and "max-" prefixes can be used: media="screen and (monochrome:2)"
  • resolution - Specifies the pixel density (dpi or dpcm) of the target display/paper, "min-" and "max-" prefixes can be used: media="print and (resolution:300dpi)"
  • scan - Specifies scanning method of a tv display, possible values are "progressive" and "interlace": media="tv and (scan:interlace)"
  • grid - Specifies if the output device is grid or bitmap, possible values are "1" for grid, and "0" otherwise: media="handheld and (grid:1)"

An <img> element also has a dimension attribute source, initially set to the element itself. The width and height attributes always define width and height of the image in pixels. Image Maps

An image map allows geometric areas on an image to be associated with hyperlinks, you can then create clickable areas on an image. The HTML <map> tag defines an image map, and the areas are defined with one or more <area> tags. An image, in the form of an <img> element, may be associated with an image map. The image is inserted using the <img> tag, the only difference from other images, is that you add a usemap attribute. The usemap value starts with a hash tag (#), followed by the name of the image map, it is used to create a relationship between the image and the image map: <map name="imagemap"> <img src="animage.jpg" alt="Image Description" usemap="#imagemap">
Then, add the clickable areas defined using an <area> element; you must also define the shape of the clickable area. You must also define coordinates to be able to place the clickable area onto the image, and can choose one of these shape values:

  • default - Defines the entire region.
  • rect - Defines a rectangular region, the coordinates come in pairs, one for each x-axis and y-axis.
  • circle - Defines a circular region, to add the circle area, first locate the coordinates of the center of the circle; then specify the radius.
  • poly - Defines a polygonal region, and contains several coordinate points, which creates a shape formed with straight lines (as a polygon). This can be used to create any shape, and coordinates come in pairs, one for each x-axis and y-axis.

map Element - <map>

Element defines a client-side image map (image with clickable areas), the element represents its children. The <map> element, in conjunction with an <img> element and any <area> element descendants, defines an image map. The areas are defined with one or more <area> element tags, that defines the clickable areas.

map Attributes - The name attribute gives the map a name so that it can be referenced - The attribute must be present. If the id attribute is also specified, both attributes must have the same value.
name (Required)
The name attribute specifies the name of the image map, it gives the map a name so that it can be referenced. The name attribute is associated and referenced with the <img> element's usemap attribute; it creates a relationship between the image and map: <map name="value">
The name attribute must be present and must have a non-empty value with no whitespace. The value must not be equal to the value of the name attribute of another map element in the same tree. If the global id attribute is also specified, both attributes must have the same value.

area Element - <area>

Element defines an area inside an image map (image with clickable areas), elements are always nested inside a <map> tag; this element has no end tag. The element represents either a hyperlink with text and a corresponding area on an image map, or a dead area on an image map. The usemap attribute in an <img> element, is associated with the <map> element's name attribute; it creates a relationship between the image and map.

area Attributes - If the global itemprop attribute is specified on an area element, then the href attribute must also be specified.
shape
The shape attribute is an enumerated attribute, and specifies the kind of shape to be created in an image map area. The shape attribute is used together with the coords attribute to specify the size, shape and placement of an area. The attribute may be omitted, its missing value default and invalid value default are the rectangle state. In the default state, area elements must not have a coords attribute - The area is the whole image: <area shape=default|rect|circle|poly> { default - Specifies the entire region. | rect - Defines a rectangular region. | circle - Defines a circular region. | poly - Defines a polygonal region. }
coords
The coords attribute specifies the coordinates for the shape of an area to be created in an image map; and must contain a valid list of floating-point numbers. The coords attribute is used together with the shape attribute to specify the size, shape and placement of an area. This gives coordinates for the shape described by the shape attribute, coordinates of the top-left corner of an area are 0,0; the following applies to each shape:
  • default - In the default state, area elements must not have a coords attribute - area is the whole image.
  • rectangle - In the rectangle state, area elements must have a coords attribute with exactly four integers. The first, must be less than the third, the second of which must be less than the fourth. The four points must represent, respectively, the distance from the left edge of the image to the left side of the rectangle. The distance from the top edge to the top side, the distance from the left edge to the right side, and distance from the top edge to the bottom side (all in pixels).
  • circle - In the circle state, <area> elements must have a coords attribute with three integers, the last of which must be non-negative. The first, must be the distance in pixels from the left edge of the image to the center of the circle. The second integer, must be the distance in pixels from the top edge of the image to the center of the circle. The third, must be the radius of the circle, again in pixels.
  • polygon - In the polygon state, area elements must have a coords attribute with at least six integers, and the number of integers must be even. Each pair of integers, must represent a coordinate given as the distances from the left and the top of the image in pixels respectively. And all the coordinates together, must represent the points of the polygon in order.
<area coords="x1,y1,x2,y2|x,y,radius|x1,y1,x2,y2,..,xn,yn"> { x1,y1,x2,y2 (shape="rect") - Specifies the coordinates of the top-left and bottom-right corner of the rectangle. | x,y,radius (shape="circle") - Specifies the coordinates of the circle center and the radius. | x1,y1,x2,y2,..,xn,yn (shape="poly") - Specifies the coordinates of the edges of the polygon, if the first and last coordinate pairs not the same, the browser will add the last coordinate pair to close the polygon. }
alt
The alt attribute provides alternative information for an image or area, if the image cannot be displayed or if the user uses a screen reader. The attribute is required if the href attribute is present, and may be left blank if another area element's (in same image map) alt attribute points to the same resource: <area alt="value">
href
The href attribute specifies the hyperlink target (address of the hyperlink) for the area, the alt attribute must be present; it specifies the text of the hyperlink. If the <area> element has no href attribute, the area represented by the element cannot be selected (and is not a hyperlink), and the alt attribute must be omitted. The target, download, ping, rel and referrerpolicy attributes must be omitted if the href attribute is not present. If the global itemprop attribute is specified on an <area> element, then the href attribute must also be specified: <area href="value"> { Possible values: Absolute URL - Points to another web site, like: href="http://www.website.com/webpage.htm") | Relative URL - Points to a file within a web site, like: href="localpage.htm" | id - Link to an element with a specified id within the page, like: href="#pageSection" | Other Protocols - Other protocols, like: https://, ftp://, mailto:, file:, etc.. | Script - A script, like href="javascript:alert('Hello');") }
target
The target attribute specifies where to open the linked document, and is only used if the href attribute is present: - area target="_blank|_self|_parent|_top|framename"> { _blank - Opens the linked document in a new window or tab. | _self - Opens the linked document in the same frame as it was clicked. | _parent - Opens the linked document in the parent frame. | _top - Opens the linked document in the full body of the window. | framename - Opens the linked document in a named iframe. }
download
The download attribute specifies that the target of the href attribute, will be downloaded when a user clicks on the hyperlink (instead of navigating to it). The value of the download attribute will be the new name of the file after it is downloaded, if the value omitted, the original filename is used. There are no restrictions on allowed values, and the browser will automatically detect the correct file extension and add it to the file: <area download="value"> - Chrome 65+ and Firefox only support same-origin download links.
ping
URLs to ping, if present, this gives the URLs of the resources that are interested in being notified if the user follows the hyperlink. The value must be a set of space-separated tokens, each of which must be a valid non-empty URL whose scheme is an HTTP(S) scheme.[]
rel
The rel attribute specifies the relationship between the current document and the linked document, is only used if the href attribute is present: <area rel=alternate|author|bookmark|help|license|next|nofollow|noreferrer|prefetch|prev|search|tag> { alternate - Links to an alternate version of the document (i.e. print page, translated or mirror). | author - Links to the author of the document. | bookmark - Permanent URL used for bookmarking. | help - Links to a help document. | license - Links to copyright information for the document. | next - The next document in a selection. | nofollow - Links to an unendorsed document, like a paid link, "nofollow" is used by Google, to specify that the Google search spider should not follow that link. | noreferrer - Specifies that the browser should not send a HTTP referer header if the user follows the hyperlink. | prefetch - Specifies that the target document should be cached. | prev - The previous document in a selection. | search - Links to a search tool for the document. | tag - A tag (keyword) for the current document. }
referrerpolicy
The referrerpolicy attribute specifies which referrer information or referrer policy to send when the user clicks on the hyperlink (for fetches initiated by the element): <area referrerpolicy="no-referrer-when-downgrade|no-referrer|origin|origin-when-cross-origin|same-origin|strict-origin-when-cross-origin|unsafe-url">> { no-referrer-when-downgrade (Default) - Sends the origin, path and query string if the protocol security level stays the same or is higher; and will send nothing to a less secure level (HTTPS to HTTP not ok). | no-referrer - No referrer information is sent. | origin - Sends the origin (scheme, host and port) of the document. | origin-when-cross-origin - Sends the origin of the document for cross-origin request, and sends the origin, path and query string for same-origin request. | same-origin - Sends a referrer for same-origin request, and sends no referrer for cross-origin request. | strict-origin-when-cross-origin - Sends the origin if the protocol security level stays the same or is higher, and will send nothing to a less secure level (HTTPS to HTTP). | unsafe-url - Sends the origin, path and query string (regardless of security) - Use this value carefully. }

The hreflang attribute specifies the language of the target URL in the area, this attribute is only used if the href attribute is set. This attribute is purely advisory, a two-letter language code specifies the language of the linked document: <area hreflang="value"> - To view all available language codes, see the Language code reference.
media
The media attribute specifies what media/device the target URL is optimized or designed for (like iPhone, speech or print media). This attribute is purely advisory, it is only used if the href attribute is present; and can accept several values: <area media="value">

  • all - (Default) Suitable for all devices.

HTML Iframes

An HTML iframe is used to display a web page within a web page, the <iframe> element specifies an inline frame. An inline frame is used to embed another document within the current HTML document. It is a good practice to always include a title attribute, this is used by screen readers to read what the content of the iframe is. An iframe can also be used as the target frame for a link, the target attribute of the link must refer to the name attribute of the iframe.

iframe element - <iframe>

The element never has fallback content, as it will create a new child navigable, regardless of whether initial contents successfully used. In legacy user agents that do not support <iframe> elements, the contents would be parsed as markup that could act as fallback content. The <iframe> element also supports dimension attributes, for cases where the embedded content has specific dimensions. It is a good practice to always include a title attribute for the <iframe>, it is used by screen readers to read out what the content of the <iframe> is.

Document metadata

HTML metadata is data about the HTML document (data about data), and is not displayed but is machine parsable. The metadata content, is content that sets up the presentation or behavior of the rest of the content. It also sets up the relationship of the document with other documents, or conveys other "out of band" information. Elements from other namespaces, whose semantics are primarily metadata-related (e.g. RDF) are also metadata content. The <head> element defines metadata about the document, and is a container for the metadata; it is placed between the <html> and <body> tags. The HTML <head> element is a container for the following elements: <base> - <link> - <meta> - <noscript> - <script> - <style> - <template> - <title>
Metadata typically defines the document title, character set, styles, scripts, and other meta information. Metadata is used by browsers for how to display content or reload a page, also by search engines for keywords and is used for other web services. To display an HTML page correctly, a web browser must also know the character set used in the page. There is also a method to let web designers take control over the viewport (the web page's visible area), through the <meta> tag.

meta Element - <meta>

Element defines metadata about an HTML document, metadata is data about data, and the tags always go inside of the <head> element. The metadata will not be displayed, but is machine parsable; it is used by browsers for how to display content or reload a page, search engines for keywords and other web services. The element is typically used to specify character set, page description, keywords, author of the document and viewport settings. It represents various kinds of metadata that cannot be expressed using the <title>, <base>, <link>, <style> and <script> elements. There is also a method to let web designers take control over the viewport (the visible area of a web page), to create a Responsive Web Design. A responsive web page will automatically adjust for different screen sizes or viewports.

meta Attributes - Exactly one of the name, http-equiv, charset and itemprop attributes must be specified. The <meta> element can represent document-level metadata with the name attribute, pragma directives with the http-equiv attribute and a file's character encoding with the charset attribute. If either name, http-equiv or itemprop is specified, then the content attribute must also be specified; otherwise, it must be omitted. If a <meta> element has no content attribute, then the value part of the metadata name-value pair is the empty string.
name
The name attribute specifies the name for the metadata, it sets the aspect of the documents metadata. Document metadata is expressed in terms of name-value pairs, the name attribute giving the name, and the content attribute (on the same element) giving the value. If a meta element has no content attribute, then the value part of the metadata name-value pair is the empty string. If the http-equiv attribute is set, the name attribute should not be set. HTML5 has also introduced a method to let web designers take control over the viewport, through the <meta> tag: <meta name=application-name|author|description|generator|keywords|viewport> { application-name - Specifies the name of the Web application that the page represents. | author - Specifies the name of the author of the document: < meta name="author" content="John Doe"> | description - Specifies a description of the web page, search engines can pick up this description to show with the results of searches: <meta name="description" content="Free Web tutorials"> | generator - Specifies one of the software packages used to generate the document, and is not used on hand-authored pages: <meta name="generator" content="FrontPage 4.0"> | keywords - Specifies a comma-separated list of keywords, relevant to the page; always specify keywords, as this is needed by search engines to catalogize the page: <meta name="keywords" content="HTML, meta tag, tag reference"> | viewport - Controls the viewport (visible area of a web page), it gives the browser instructions on how to control the page's dimensions and scaling. The viewport varies with the device, and will be smaller on a mobile phone than on a computer screen; you should include the <meta> viewport element in all your web pages. Setting the viewport to make your website look good on all devices: <meta name="viewport" content="width=device-width, initial-scale=1.0"> - The initial-scale=1.0, sets the initial zoom level when the page is first loaded by the browser. The width=device-width, sets the width of the page to follow the screen-width of the device (will vary depending on the device). }
http-equiv
The http-equiv attribute provides an HTTP header, for the information/value of the content attribute; it can be used to simulate an HTTP response header: <meta http-equiv=content-security-policy|content-type|default-style|refresh> { content-security-policy - Specifies a content policy for the document: <meta http-equiv="content-security-policy" content="default-src 'self'"> | content-type - Specifies the character encoding for the document: <meta http-equiv="content-type" content="text/html; charset=UTF-8"> | default-style - Specified the preferred style sheet to use: <meta http-equiv="default-style" content="the document's preferred stylesheet"> - The value of the content attribute, must match the value of the title attribute on a link element in the same document. Or, it must match the value of the title attribute on a <style> element in the same document. | refresh - Defines a time interval for the document to refresh itself, to refresh the document every 30 seconds: <meta http-equiv="refresh" content="30"> - The value refresh should be used carefully, as it takes the control of a page away from the user, and will cause a failure in W3C's Web Content Accessibility Guidelines. }
content
The content attribute gives the value associated with the http-equiv or name attribute (pragma directive or document metadata). If a meta element has no content attribute, then the value part of the metadata name-value pair is the empty string: <meta content="value">
charset
The charset attribute specifies the character encoding (character set) used by the document, its value must be an ASCII case-insensitive match for the string "utf-8". There must not be more than one <meta> element with a charset attribute per document. The HTML5 specification encourages web developers to use the UTF-8 character set, which covers almost all of the characters and symbols in the world: <meta charset="value">
media
The media attribute says which media the metadata applies to (the applicable media), the value must be a valid media query list. Unless the name is theme-color, the media attribute has no effect on the processing model and must not be used by authors. []

Element defines the relationship between a document, and an external resource; it allows authors to link their document to other resources. It is an empty element, and contains attributes only; it is most often used to link external style sheets or to add favicon (icons) to a website. Two categories of links can be created using the <link> element, links to external resources and hyperlinks. One link element can create multiple links, of which some might be external resource links, and some might be hyperlinks.

A link element must also have either a rel or itemprop attribute, but not both. The types of link indicated (the relationships) are given by the value of the rel attribute, if used, the element can only sometimes be used in the page body. When used with the itemprop attribute, the element can be used in the head element and body of the page. Hyperlinks created with the rel attribute apply to the whole document, this contrasts with the rel attribute of <a> and <area> elements.
href
The href attribute specifies the location (URL) of the external resource or linked document, its value must be a valid non-empty URL potentially surrounded by spaces. One or both of the href or imagesrcset attributes must be present: link href="value"> { Possible values: Absolute URL - Points to another web site, like: href="http://www.website.com/stylesheet.css" | Relative URL - Points to a file within a web site, like: href="/local/stylesheet.css" }

HTML JavaScript

JavaScript is the Programming Language for the Web, it can update or change both HTML and CSS; JavaScript can also calculate, manipulate and validate data. When a web page is loaded, the browser creates a Document Object Model of the page, the HTML DOM model is constructed as a tree of Objects; and is an API (Application Programming Interface) for JavaScript. Scripts allow authors to add interactivity to their documents, the HTML <script> tag is used to define a client-side script (JavaScript). Script-supporting elements are those that do not represent anything themselves (are not rendered), but are used to support scripts. The <script> element either contains script statements, or it points to an external script file through the src attribute. Common uses for JavaScript are image manipulation, form validation and dynamic changes of content (such as style or attributes). Through scripting and using other mechanisms, the values of attributes, text and the structure of a document may change while a user agent is processing it. User agents update their presentation of the document as this occurs. Servers should use text/javascript for JavaScript resources, in accordance with Updates to ECMAScript Media Types, ECMAScript 5 is also known as JavaScript 5 or ECMAScript 2009.

When you want to access HTML elements with JavaScript, you have to find the elements first; there are a couple of ways to do this:

  • By: id.
  • By: tag name
  • By: class name
  • By: CSS selectors
  • By: object collections
JavaScript most often uses the document.getElementById() method, this example writes "Hello JavaScript!" into an HTML element: <script> document.getElementById("demo").innerHTML = "Hello JavaScript!"; </script> - The global id attribute, can also be used by JavaScript to perform tasks for that specific element.
You can also use the comment tag to hide scripts, or comment on code; two forward slashes (//) at the end of a comment line, is the JavaScript comment symbol: <script type="text/javascript"> </script>
Authors are encouraged to make their applications degrade gracefully, in the absence of scripting support. Look at the <noscript> element, for users that have disabled scripts, or a browser that doesn't support client-side scripting.

The crossorigin attribute is a CORS settings attribute, it sets the mode of the request to an HTTP CORS Request. Element defines alternate content/markup to be displayed for users that do not support client-side scripts, or that have disabled scripts in their browser. The <noscript> element can be used in both the <head> and <body> elements, when inside <head>, it can only contain <link>, <style> and <meta> elements: <noscript>Your browser does not support JavaScript, or JavaScript is disabled (check your browser settings)! </noscript>

template Element - <template>

Element is used to declare fragments of HTML that can be cloned and inserted in the document by script, it defines a container for content that should be hidden when page loads. You can use the tag if you have HTML code you want to use over and over again, but not until you ask for it.

slot Element - <slot>

Element defines a slot, it represents its assigned nodes, if any, and its contents otherwise; it is typically used in a shadow tree.

slot Attributes
name
The name attribute may contain any string value, it represents a slot's name (name of shadow tree slot). The name attribute is used to assign slots to other elements, it creates a named slot to which any element is assigned. If that element has a slot attribute whose value matches that name attribute's value. And the <slot> element is a child of the shadow tree, whose root's host has that corresponding slot attribute value.

progress Element - <progress>

Element represents the completion progress of a task, use the tag in conjunction with JavaScript, to display the progress of a task. The progress is either indeterminate, indicating progress being made but it is not clear how much work remains. Or the progress is a number in the range zero to a maximum, giving the fraction of work that has so far been completed. Always add <label> tag for best accessibility practices.

progress Attributes - Authors encouraged to include the current and maximum attribute values as text inside the element, so the progress is made available to users of legacy user agents.
value
The value attribute specifies how much of the task has been completed, as a floating point number that specifies the task completion. If present, it must have a value equal to or greater than zero, and less than or equal to the value of the max attribute, if present, or 1.0 otherwise: <progress value="value"> - To make a determinate progress bar, add a value attribute with the current progress, and to make an indeterminate progress bar, remove the value attribute.
max
The max attribute specifies how much work the task requires in total (the upper bound of range), it must have a floating point number value greater than zero; the default is 1: <progress max="value">

HTML Plug-ins

Plug-ins are computer programs that extend the standard functionality of the browser. The term plugin refers to an implementation-defined set of content handlers, used by the user agent; and that can take part in the user agent's rendering of a Document object.
Plug-ins were designed to be used for many different purposes:

  • To run Java applets - Most browsers no longer support Java Applets and Plug-ins.
  • To display maps
  • To scan for viruses
  • To run Microsoft ActiveX controls - ActiveX controls are no longer supported in any browsers.
  • To verify a bank id
  • To display Flash movies - The support for Shockwave Flash has also been turned off in modern browsers.
Typically such content handlers are provided by third parties, though a user agent can also designate built-in content handlers as plugins.

The external resource can be a web page, a picture, a media player or a plug-in application. Plug-ins are computer programs that extend the standard functionality of the browser, and have been used for many different purposes. The <object> tag was originally designed to embed browser Plug-ins (like Java applets, PDF readers and Flash Players) in web pages. It can also be used to include images, video and HTML in HTML if you like. To embed a picture, it is better to use the <img> tag, for video or audio, it is better to use the <video> and <audio> tags. Element adds a machine-readable translation of a given content, and represents its contents; along with a machine-readable form of those contents in the value attribute. The element can also, be used in conjunction with scripts in the page, for when a script has a literal value to store alongside a human-readable value. value

time Element - <time>

Element defines a specific time (or datetime), along with a machine-readable form of those contents in the datetime attribute. The kind of content is limited to various kinds of dates, times, time-zone offsets and durations. The format used "on the wire" (i.e. HTML markup or form submissions), is intended to be computer-readable and consistent; irrespective of a user's locale. A date for instance, is always written in the format "YYYY-MM-DD", as in "2003-02-01", some users might see this format; others may see "01.02.2003" or "February 1, 2003".

HTML Graphics - Canvas

The HTML <canvas> element can be used to draw graphics on a web page. A canvas is a rectangular area on an HTML page, by default, a canvas has no border and no content. Canvas has several methods for drawing paths, boxes, circles, text and adding images. The <canvas> element is only a container for graphics, you must use JavaScript to actually draw the graphics.

canvas Element - <canvas>

Element is used to draw graphics via scripting, it is only a container for graphics; you must use JavaScript to actually draw graphics. The <canvas> element provides scripts with a resolution-dependent bitmap canvas, which can be used for rendering graphs, game graphics, art, or other visual images. Canvas has several methods for drawing paths, boxes, circles, text and adding images. Authors must also provide content that conveys essentially the same function or purpose as the canvas's bitmap. This content may be placed as content of the <canvas> element, this content is the element's fallback content. Any text inside the element will be displayed in browsers when JavaScript is disabled, and also in browsers that do not support the <canvas> tag. Always specify an id attribute to be referred to in a script, and to add a border, use the style attribute.

- The canvas element has two attributes to control the size of the element's bitmap, width and height; always specify a width and height attribute.
width
The width attribute specifies the width (horizontal dimension) of the <canvas> element in pixels, the default value is 300: <canvas width="value">
height
The height attribute specifies the height (vertical dimension) of the <canvas> element in pixels, the default value is 150: <canvas height="value">

HTML Graphics - Scalable Vector Graphics (SVG)

SVG stands for Scalable Vector Graphics, it defines vector-based graphics in XML format. SVG is used to define graphics for the Web, and is a W3C recommendation. The HTML <svg> element is a container for SVG graphics, and has several methods for drawing paths, boxes, circles, text and graphic images.

svg Element - <svg>

The <svg> tag defines a container for SVG graphics, and has several methods for drawing paths, boxes, circles, text and graphic images: <svg width="200" height="100"> SVG Content </svg>

Differences Between SVG and Canvas

SVG is a language for describing 2D graphics in XML, while Canvas draws 2D graphics, with a JavaScript. Since SVG is XML based, this means every element is available within the SVG DOM; you can then attach JavaScript event handlers for an element. Each drawn shape is remembered as an object, and if the attributes of an SVG object are changed, the browser can automatically re-render the shape.
Canvas is rendered pixel by pixel, once the graphic is drawn, it is forgotten by the browser. If its position should be changed, the entire scene needs to be redrawn, including any objects that might have been covered by the graphic.
